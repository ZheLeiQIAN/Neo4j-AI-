## 一、从 Word2Vec 到图嵌入：DeepWalk 的核心洞察

DeepWalk 是图嵌入领域的奠基性工作（Perozzi 等，2014）。它的核心洞察简洁而深刻：**既然 Word2Vec 能从词在句子中的共现关系学习词向量，那么把图上的随机游走序列当作"句子"、把节点当作"词"，同样的算法就能从节点在游走中的共现关系学习节点向量。**

这一洞察之所以成立，是因为两者在数学结构上同构：句子是词的线性序列，游走是节点的线性序列；词的上下文由句子中相邻的词定义，节点的上下文由游走中相邻的节点定义。Word2Vec 的 Skip-gram 模型最大化"给定中心词预测上下文词"的对数似然，DeepWalk 直接复用这一目标函数，只是把输入从自然语言语料换成了随机游走语料。

DeepWalk 的算法流程可以概括为三步：

1. **随机游走采样**（Random Walk Sampling）：对图中每个节点，从该节点出发做若干次固定长度的随机游走（完全随机，每一步等概率走到任一邻居），得到大量游走序列；
2. **Skip-gram 训练**（Skip-gram Training）：用游走序列训练 Word2Vec 的 Skip-gram 模型，学习每个节点的向量表示；
3. **输出嵌入**：训练收敛后，每个节点对应一个稠密向量。

DeepWalk 的随机游走是完全无偏的——每一步从当前节点均匀随机地走向任一邻居。这种无偏性使 DeepWalk 隐式地倾向于捕捉同质性（Homophily）：相邻的节点倾向于在游走中频繁共现，从而在嵌入空间中靠近。但 DeepWalk 无法灵活控制是更看重同质性还是结构等价（Structural Equivalence），这正是 Node2Vec 试图改进的地方。

## 二、Node2Vec：在 BFS 与 DFS 之间游走

Node2Vec（Grover & Leskovec，2016）在 DeepWalk 的基础上引入了一个关键创新：**带偏置的二阶随机游走**（Biased Second-order Random Walk）。通过两个参数 p 与 q，Node2Vec 可以在广度优先搜索（Breadth-First Search，BFS）与深度优先搜索（Depth-First Search，DFS）两种游走策略之间连续插值，从而灵活平衡两种不同的结构相似性。

### 2.1 两种结构相似性

理解 Node2Vec，首先要区分图中的两种结构相似性：

- **同质性（Homophily）**：直接相连的节点倾向于相似。在社交网络中，好友往往有相似的兴趣；在引文网络中，直接引用的论文往往属于相近领域。同质性由 DFS 策略捕捉——深游走会探索远处的节点，让同一"社区"的节点嵌入靠近。
- **结构等价性（Structural Equivalence）**：在网络中扮演相似角色的节点倾向于相似，即使它们相距很远。例如，两个不同公司里的"桥梁型"员工（连接两个社群的人）在结构上等价。结构等价由 BFS 策略捕捉——浅游走会探索局部邻域，让结构角色相似的节点嵌入靠近。

DeepWalk 的无偏游走无法区分这两种相似性。Node2Vec 的贡献在于：用参数化的偏置游走，让用户根据任务需要选择"看同质性"还是"看结构等价"。

### 2.2 二阶随机游走的转移概率

Node2Vec 的游走不是一阶的（只依赖当前节点），而是二阶的——转移概率取决于"上一节点 t、当前节点 v、候选下一节点 x"三者之间的关系。设游走刚从 t 走到 v，现在要从 v 走到 x，转移概率定义如下：

- 如果 x = t（回到上一节点），概率权重为 **1/p**；
- 如果 x 与 t 相邻（即 v 和 t 有共同邻居 x，走一步"侧向"），概率权重为 **1**；
- 如果 x 与 t 不相邻（向前走远），概率权重为 **1/q**。

这里 p 称为**返回参数**（Return Parameter），q 称为**入度/出度参数**（In-Out Parameter）。

- **p 大**：不倾向于回到上一节点，鼓励游走走远 → 偏向 DFS → 偏向同质性；
- **q 大**：不倾向于走向远处，鼓励游走留在局部 → 偏向 BFS → 偏向结构等价；
- **p = q = 1**：退化为无偏随机游走，等价于 DeepWalk。

### 2.3 参数 p 与 q 的语义与调优建议

| 参数 | 作用 | 增大效果 | 减小效果 | 推荐取值场景 |
|------|------|---------|---------|------------|
| p（返回参数） | 控制回到上一节点的倾向 | 鼓励探索远处，减少回访 | 频繁回访，游走更局部 | 同质性任务 p 小；结构等价任务 p 大 |
| q（入出参数） | 控制走向远处 vs 局部 | 偏向 BFS，捕捉结构等价 | 偏向 DFS，捕捉同质性 | 同质性任务 q 小；结构等价任务 q 大 |

调优建议：

- **节点分类任务**（同一社区同类标签）：通常同质性更重要，建议 p=1, q=0.5（偏 DFS）；
- **链路预测任务**（预测边存在）：同质性与结构等价都有用，建议 p=1, q=1（均衡）或小范围网格搜索；
- **结构角色发现**（如桥节点、枢纽节点识别）：结构等价更重要，建议 p=1, q=2（偏 BFS）。

实践中的标准做法是对 (p, q) 做小规模网格搜索（如 p ∈ {0.25, 0.5, 1, 2, 4}，q ∈ {0.25, 0.5, 1, 2, 4}），以下游任务指标为准选择最优组合。

## 三、算法原理详解

### 3.1 目标函数

Node2Vec 的目标函数与 DeepWalk（及 Word2Vec Skip-gram）一致：最大化给定节点 u 时其上下文节点集合 N(u) 出现的对数似然。

$$\max_f \sum_{u \in V} \left[ -\log Z_u + \sum_{n \in N_S(u)} \log \sigma(f(n) \cdot f(u)) \right]$$

其中 f(u) 是节点 u 的嵌入向量，N_S(u) 是通过策略 S 采样得到的 u 的邻域（上下文节点集合），σ 是 sigmoid 函数，Z_u 是配分函数（归一化项），通常用负采样（Negative Sampling）近似。

### 3.2 采样策略

与 DeepWalk 不同，Node2Vec 的邻域 N_S(u) 不是通过无偏游走得到，而是通过上述二阶偏置游走策略 S 采样得到。这一策略由参数 (p, q) 定义。采样的核心步骤：

1. 对每个节点 u，从 u 出发启动 r 条游走，每条游走长度 l；
2. 游走的第 0 步在 u，第 1 步随机走向 u 的任一邻居；
3. 从第 2 步起，按二阶转移概率选择下一节点；
4. 收集所有游走序列作为"语料"。

### 3.3 优化

采样得到的游走序列直接喂给 Word2Vec 的 Skip-gram 训练。优化采用随机梯度上升（SGA）配合负采样，以降低计算成本。训练收敛后，每个节点对应一个 d 维向量（d 由 embeddingDimension 参数指定）。

从理论角度，DeepWalk/Node2Vec 的随机游走采样 + Skip-gram 训练，在数学上等价于对节点共现矩阵（近似 PMI 矩阵）做隐式矩阵分解。这一等价关系是后来 FastRP 等显式矩阵分解方法的理论基础。

## 四、Neo4j GDS 中的 Node2Vec 调用示例

Neo4j GDS 提供了 Node2Vec 算法的工程实现。以下是一个完整的调用示例，展示从图投影到嵌入写回的流程。

```cypher
// 1. 创建图投影（无向图）
CALL gds.graph.project(
  'citationGraph',
  'Paper',
  { CITES: { orientation: 'UNDIRECTED' } }
);

// 2. 运行 Node2Vec 并将嵌入写回节点属性
CALL gds.node2vec.write(
  'citationGraph',
  {
    writeProperty: 'n2vEmbedding',
    embeddingDimension: 128,
    walkLength: 80,         // 单条游走长度
    walksPerNode: 10,       // 每节点游走次数
    returnParameter: 1.0,   // 参数 p
    inOutParameter: 0.5,    // 参数 q（偏 DFS，强调同质性）
    iterations: 10,         // Skip-gram 训练轮数
    windowSize: 10,         // 上下文窗口大小
    concurrency: 4
  }
) YIELD nodePropertiesWritten, computeMillis;

// 3. 查询某篇论文的嵌入向量
MATCH (p:Paper {title: 'Graph Neural Networks'})
RETURN p.n2vEmbedding AS embedding;
```

参数说明：

| 参数 | 含义 | 推荐值 |
|------|------|-------|
| embeddingDimension | 嵌入维度 | 128（默认），大型图可用 256 |
| walkLength | 游走长度 | 80（经典值），越大覆盖越远 |
| walksPerNode | 每节点游走次数 | 10，越大越稳定但越慢 |
| returnParameter | 参数 p | 1.0（默认），结构任务可调大 |
| inOutParameter | 参数 q | 1.0（默认），同质任务可调小 |
| iterations | Skip-gram 训练轮数 | 10 |
| windowSize | 上下文窗口 | 10 |

GDS 的 Node2Vec 默认输出流模式（stream）、写回模式（write）与统计模式（stats）三种执行模式。生产中常用 write 模式把嵌入持久化到节点属性，再配合向量索引做相似度检索。

## 五、DeepWalk 与 Node2Vec 的对比

| 对比维度 | DeepWalk | Node2Vec |
|---------|----------|----------|
| 游走策略 | 完全无偏随机游走 | 二阶偏置随机游走 |
| 参数 | 无偏置参数 | p、q 两个参数 |
| 结构相似性 | 隐式偏向同质性 | 可灵活平衡同质性与结构等价 |
| 灵活性 | 低 | 高 |
| 调参成本 | 低 | 中（需调 p、q） |
| 理论地位 | 开创性工作 | 改进与泛化 |

可以说，DeepWalk 是 Node2Vec 在 p=q=1 时的特例。GDS 中并未单独提供 DeepWalk 算法，而是将其作为 Node2Vec 的默认配置（p=1, q=1）实现。因此实践中使用 GDS 的 node2vec 过程即可同时覆盖两种方法。

## 六、图像领域的应用：图像节点向量化表示

虽然 Node2Vec 起源于社交网络分析，但它在图像领域同样有重要应用，前提是把图像数据转化为图结构。典型的应用场景包括：

### 6.1 场景图上的节点嵌入

在视觉推理（Visual Reasoning）任务中，图像被解析为场景图（Scene Graph）：图中检测到的物体为节点，物体间的空间与语义关系（"在...上""包含""属于...颜色"）为边。在场景图上运行 Node2Vec，可以得到每个物体节点的结构嵌入，编码它在场景中的角色。这种结构嵌入可与物体的视觉特征（CNN 特征向量）拼接，用于场景图生成、视觉问答（Visual Question Answering，VQA）等任务。

### 6.2 图像相似度图上的嵌入

在图像检索场景中，可以先构建一张"图像相似度图"：每张图像为节点，用预训练 CNN 提取视觉特征，特征相似的图像之间连边（k 近邻图）。在这张图上运行 Node2Vec，得到的图像嵌入既包含视觉相似性（来自 CNN 特征），又包含结构相似性（来自图的拓扑），比纯视觉特征更具区分力。

### 6.3 超像素图上的嵌入

在图像分割任务中，图像被划分为超像素（Superpixel），相邻超像素之间连边构成超像素图。Node2Vec 可以为每个超像素生成嵌入，用于聚类分割或语义分割。这是把 CNN 不可直接处理的"区域关系"用图嵌入表达的典型例子。

## 七、与图书情报领域的关联：引文网络节点嵌入

Node2Vec 在图书情报领域最自然的应用场景是引文网络（Citation Network）。

### 7.1 引文网络嵌入的意义

引文网络中，论文通过引用关系相互连接。在这张图上运行 Node2Vec，每篇论文得到一个结构嵌入向量。这个向量编码了论文在引用网络中的"位置特征"：

- 同质性：互相引用或被同一批论文引用的论文（同领域、同主题），嵌入向量相近；
- 结构等价：在引用网络中扮演相似角色（如都是某领域的枢纽综述、都是被冷落的边缘论文），嵌入向量相近。

这种结构嵌入与基于文本的嵌入（TF-IDF、BERT）互补——前者捕捉"引用结构上的相似"，后者捕捉"内容上的相似"。两者融合可以显著提升论文检索、推荐、分类的效果。

### 7.2 具体应用

- **相关论文推荐**：给定一篇论文，用其 Node2Vec 向量在向量索引中检索最近邻，推荐引用结构相似的论文。这比单纯的关键词匹配更能发现"方法相同但表述不同"的相关工作；
- **研究前沿识别**：对论文嵌入做聚类（如 Louvain 或 K-Means），每个簇对应一个研究社区。新论文的簇归属可以揭示研究趋势；
- **论文影响力预测**：把 Node2Vec 嵌入作为特征，结合 PageRank、被引次数等，训练回归模型预测论文的未来影响力；
- **跨学科论文发现**：在嵌入空间中位于多个簇交界处的论文，往往是跨学科工作。可用嵌入的"社区归属度"识别这类论文。

### 7.3 参数选择的领域考量

在引文网络中，同质性通常比结构等价更重要——引用同一篇论文的论文大概率属于同一领域。因此实践中常取 q < 1（偏 DFS，强调同质性）。但若任务是识别"综述型论文"（结构等价角色），则应取 q > 1。

需要注意的是，Node2Vec 是直推式方法——引文网络新增论文后，必须重新运行整个嵌入流程。对于频繁更新的引文库，这是显著的工程成本，GraphSAGE 等归纳式方法可缓解此问题。

## 八、小结

DeepWalk 与 Node2Vec 是图嵌入领域的经典方法，它们通过"随机游走 + Word2Vec"的巧妙类比，把 NLP 的成熟工具引入图学习。Node2Vec 的参数化偏置游走使其成为迄今最灵活的随机游走嵌入方法。理解它们的原理与参数语义，是掌握后续 FastRP、GraphSAGE 等方法的基础。下一章将介绍如何用矩阵分解思路（FastRP）在不牺牲质量的前提下大幅提升嵌入速度。

## 一、中心性概念

中心性（Centrality）是图论中用于衡量节点重要性的指标族。在一个网络中，不同的节点扮演不同的角色——有的节点是"枢纽"，连接着大量其他节点；有的节点是"桥梁"，连接着不同的社区；有的节点虽然连接不多，但连接的都是重要节点。中心性算法就是用数值化的方式量化这些"重要性"。

理解中心性的关键在于：**"重要性"没有唯一标准**。不同的应用场景对重要性的定义不同，因此产生了多种中心性指标。选择哪种中心性，取决于你要回答什么问题。

| 问题 | 对应的中心性 | 直觉含义 |
|------|------------|---------|
| 谁的连接最多？ | 度中心性 | 人气王 |
| 谁被重要的人连接？ | PageRank / 特征向量 | 名人效应 |
| 谁是信息桥梁？ | 介数中心性 | 中间人 |
| 谁能最快到达所有人？ | 接近中心性 | 广播站 |

在图书情报领域，中心性算法有着直接的应用价值。引文网络中，高中心性的论文就是具有高影响力的论文；合作网络中，高中心性的学者就是学术核心。传统的文献计量指标（被引频次、影响因子）本质上只度量了"度中心性"，而 GDS 提供的多种中心性算法能从更丰富的维度评估影响力。

## 二、PageRank

PageRank 是最著名的中心性算法，由 Google 创始人 Larry Page 和 Sergey Brin 于 1998 年提出，最初用于网页排名。它的核心思想是"链接投票"：一个节点的重要性取决于指向它的节点的数量和质量。

### 2.1 算法原理

PageRank 基于以下假设：

1. **链接即投票**：如果节点 A 指向节点 B，相当于 A 给 B 投了一票；
2. **票有权重**：来自高重要性节点的票，权重更高；
3. **均分原则**：一个节点的投票权被均分给它指向的所有节点；
4. **随机游走**：以概率 `1 - dampingFactor` 随机跳转到任意节点，模拟用户随机浏览行为。

数学上，PageRank 是一个迭代算法，每次迭代更新所有节点的分数，直到收敛：

```
PR(A) = (1 - d) / N + d * Σ [ PR(Ti) / C(Ti) ]
```

其中 `d` 是阻尼因子（Damping Factor，通常为 0.85），`N` 是节点总数，`Ti` 是指向 A 的节点，`C(Ti)` 是 Ti 的出度。

### 2.2 关键参数

| 参数 | 英文 | 默认值 | 说明 |
|------|------|--------|------|
| `dampingFactor` | Damping Factor | 0.85 | 随机跳转概率的补数 |
| `maxIterations` | Max Iterations | 20 | 最大迭代次数 |
| `tolerance` | Tolerance | 0.0000001 | 收敛阈值 |
| `relationshipWeightProperty` | Relationship Weight | 无 | 关系权重属性 |

### 2.3 Cypher 调用示例

```cypher
// 前提：已创建图投影
CALL gds.graph.project('citationGraph', 'Paper', 'CITES');

// stream 模式：查看 PageRank 排名
CALL gds.pageRank.stream('citationGraph', {
    dampingFactor: 0.85,
    maxIterations: 100
})
YIELD nodeId, score
RETURN gds.util.asNode(nodeId).title AS paper, score
ORDER BY score DESC LIMIT 10;

// write 模式：将 PageRank 写回数据库
CALL gds.pageRank.write('citationGraph', {
    writeProperty: 'pagerank',
    dampingFactor: 0.85
})
YIELD nodePropertiesWritten, ranIterations;

// 清理
CALL gds.graph.drop('citationGraph');
```

## 三、Betweenness Centrality（介数中心性）

介数中心性（Betweenness Centrality）衡量一个节点作为"信息桥梁"的程度。具体而言，它计算通过某节点的最短路径数量占所有最短路径的比例。

### 3.1 算法原理

对于节点 v，其介数中心性定义为：

```
BC(v) = Σ [ σ(st, v) / σ(st) ]
```

其中 `σ(st)` 是从节点 s 到节点 t 的最短路径总数，`σ(st, v)` 是这些最短路径中经过节点 v 的数量。

直观理解：如果删除某个节点会导致大量节点对之间的最短路径变长或断裂，那么该节点的介数中心性就高。这类节点是网络中的"瓶颈"或"桥梁"。

### 3.2 Cypher 调用示例

```cypher
CALL gds.graph.project('coauthorGraph', 'Author', {
    CO_AUTHOR: {orientation: 'UNDIRECTED'}
});

// stream 模式：识别学术桥梁人物
CALL gds.betweenness.stream('coauthorGraph')
YIELD nodeId, score
RETURN gds.util.asNode(nodeId).name AS author, score
ORDER BY score DESC LIMIT 10;

// write 模式
CALL gds.betweenness.write('coauthorGraph', {
    writeProperty: 'betweenness'
});

CALL gds.graph.drop('coauthorGraph');
```

介数中心性的计算复杂度较高（O(nm)），GDS 提供了基于随机采样的近似版本 `gds.betweenness.stream` 配合 `samplingSize` 参数，适用于大规模图。

## 四、Closeness Centrality（接近中心性）

接近中心性（Closeness Centrality）衡量一个节点到所有其他节点的平均距离的倒数。距离越短，接近中心性越高，意味着该节点能最快地"到达"网络中的其他节点。

### 4.1 算法原理

```
CC(v) = (N - 1) / Σ d(v, u)
```

其中 `d(v, u)` 是节点 v 到节点 u 的最短距离，`N` 是节点总数。

接近中心性高的节点适合作为信息广播的起点，因为它能以最少的跳数触达全网。

### 4.2 Cypher 调用示例

```cypher
CALL gds.graph.project('citationGraph', 'Paper', {
    CITES: {orientation: 'UNDIRECTED'}
});

// stream 模式
CALL gds.closeness.stream('citationGraph')
YIELD nodeId, score
RETURN gds.util.asNode(nodeId).title AS paper, score
ORDER BY score DESC LIMIT 10;

CALL gds.graph.drop('citationGraph');
```

## 五、Degree Centrality（度中心性）

度中心性（Degree Centrality）是最简单的中心性指标，就是节点的度数（连接数）。在有向图中，分为入度（In-Degree）和出度（Out-Degree）。

### 5.1 算法原理

```
DC(v) = deg(v)
```

虽然简单，但度中心性在很多场景下依然有效。引文网络中，论文的入度就是被引次数，这是最直观的影响力指标。

### 5.2 Cypher 调用示例

```cypher
CALL gds.graph.project('citationGraph', 'Paper', 'CITES');

// 入度（被引次数）
CALL gds.degree.stream('citationGraph', {
    orientation: 'REVERSE'           // 反转方向计算入度
})
YIELD nodeId, score
RETURN gds.util.asNode(nodeId).title AS paper, score AS in_degree
ORDER BY in_degree DESC LIMIT 10;

CALL gds.graph.drop('citationGraph');
```

## 六、Eigenvector Centrality（特征向量中心性）

特征向量中心性（Eigenvector Centrality）与 PageRank 类似，都考虑了邻居的重要性，但它不进行"均分"——一个节点的分数正比于其邻居分数之和。

### 6.1 算法原理

```
EC(v) = (1/λ) * Σ EC(u)
```

其中 `λ` 是图邻接矩阵的最大特征值。直观上，连接到一个重要节点比连接到多个不重要节点更有价值。

### 6.2 PageRank 与 Eigenvector 对比

| 对比维度 | PageRank | Eigenvector Centrality |
|---------|---------|----------------------|
| 投票分配 | 均分给出度邻居 | 不均分 |
| 随机跳转 | 有（阻尼因子） | 无 |
| 收敛性 | 更稳定 | 可能不收敛 |
| 适用场景 | 有向图 | 无向图 |
| 计算成本 | 较低 | 较高 |

### 6.3 Cypher 调用示例

```cypher
CALL gds.graph.project('coauthorGraph', 'Author', {
    CO_AUTHOR: {orientation: 'UNDIRECTED'}
});

CALL gds.eigenvector.stream('coauthorGraph')
YIELD nodeId, score
RETURN gds.util.asNode(nodeId).name AS author, score
ORDER BY score DESC LIMIT 10;

CALL gds.graph.drop('coauthorGraph');
```

## 七、图像领域应用

中心性算法在图像知识图谱分析中有着丰富的应用场景。以下是几个典型用例。

### 7.1 识别关键图像

在图像相似网络中，PageRank 可以识别"枢纽图像"——这些图像与许多其他图像相似，是图像数据集中的代表性样本。

```cypher
// 场景：在图像相似网络中识别代表性图像
CALL gds.graph.project('imageSim', 'Image', {
    SIMILAR_TO: {
        orientation: 'UNDIRECTED',
        properties: ['score']
    }
});

CALL gds.pageRank.write('imageSim', {
    writeProperty: 'imageImportance',
    relationshipWeightProperty: 'score'
})
YIELD nodePropertiesWritten;

// 查看最重要的图像
MATCH (img:Image)
RETURN img.filename, img.imageImportance
ORDER BY img.imageImportance DESC LIMIT 5;

CALL gds.graph.drop('imageSim');
```

### 7.2 识别核心物体

在物体关系网络中，介数中心性可以识别"桥梁物体"——这些物体连接不同的场景，是跨场景推理的关键。

```cypher
// 场景：识别物体关系网络中的桥梁物体
CALL gds.graph.project('objectNet', 'Object', {
    HOLDING: {orientation: 'UNDIRECTED'},
    SITTING_ON: {orientation: 'UNDIRECTED'},
    LEFT_OF: {orientation: 'UNDIRECTED'}
});

CALL gds.betweenness.write('objectNet', {
    writeProperty: 'bridgeScore'
});

MATCH (o:Object)
RETURN o.object_id, o.category, o.bridgeScore
ORDER BY o.bridgeScore DESC LIMIT 5;

CALL gds.graph.drop('objectNet');
```

### 7.3 应用场景汇总

| 中心性算法 | 图像领域应用 | 价值 |
|-----------|------------|------|
| PageRank | 识别代表性图像、核心物体 | 发现数据集中的枢纽节点 |
| Betweenness | 识别场景桥梁物体 | 跨场景关联发现 |
| Closeness | 识别中心场景 | 信息传播起点选择 |
| Degree | 识别高频物体 | 热门物体统计 |
| Eigenvector | 识别高质量图像群 | 质量加权评估 |

## 八、各算法对比表

下表系统对比了五种中心性算法的核心特征，帮助读者根据场景选择合适的算法。

| 算法 | 核心思想 | 时间复杂度 | 是否考虑邻居质量 | 适用图类型 | 典型应用 |
|------|---------|-----------|---------------|-----------|---------|
| Degree | 连接数量 | O(n) | 否 | 有向/无向 | 被引次数统计 |
| PageRank | 链接投票（均分） | O(iterations * m) | 是（均分） | 有向为主 | 论文影响力排名 |
| Eigenvector | 邻居分数之和 | O(iterations * m) | 是（不均分） | 无向为主 | 学者影响力 |
| Betweenness | 最短路径桥梁 | O(nm) | 否 | 有向/无向 | 信息桥梁识别 |
| Closeness | 平均最短距离 | O(nm) | 否 | 连通图 | 广播效率评估 |

## 九、与图书情报领域的关联

中心性算法与图书情报领域的影响力评估方法高度契合，可以说是文献计量学的"图论升级版"。

| 中心性算法 | 传统 LIS 指标 | 升级价值 |
|-----------|-------------|---------|
| Degree Centrality | 被引频次 | 完全等价 |
| PageRank | 特征因子（Eigenfactor） | 算法基础相同 |
| Eigenvector | 权重影响因子 | 考虑引用方质量 |
| Betweenness | 学科桥梁识别 | 量化跨学科性 |
| Closeness | 知识传播效率 | 量化扩散能力 |

特别值得一提的是，期刊特征因子（Eigenfactor）和 SCImago 期刊排名（SJR）都基于 PageRank 算法，是图算法在图书情报领域成功应用的典范。GDS 让研究者能够直接在自己的引文网络上运行 PageRank，而不依赖第三方数据库的预计算结果。

一个具体的研究示例：对某学科领域的引文网络运行 PageRank，可以得到该领域内论文的"结构影响力"排名，这与单纯按被引次数排名的结果往往不同——被少数高影响力论文引用的论文，其 PageRank 可能高于被大量低影响力论文引用的论文。

## 小结

本章系统介绍了 GDS 提供的五种中心性算法：PageRank（基于链接投票）、Betweenness（信息桥梁）、Closeness（广播效率）、Degree（连接数量）、Eigenvector（邻居质量加权）。每种算法从不同维度量化节点重要性，适用于不同的分析场景。在图像领域，中心性算法可识别关键图像和核心物体；在图书情报领域，它们是传统影响力指标的图论升级。

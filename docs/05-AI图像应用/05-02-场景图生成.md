## 一、场景图定义

### 1.1 什么是场景图

场景图（Scene Graph）是一种对图像内容进行结构化语义表示的数据结构。它将一幅图像中所有可识别的物体及其相互关系，组织为一个有向图：节点表示物体（Object），边表示物体间的关系（Predicate），每个物体节点还可附带属性（Attribute）。场景图的本质是把"一幅图像"翻译为"一段结构化知识"。

一个典型的场景图实例可以这样描述：一幅"人在骑马"的图像，对应场景图包含两个物体节点（人、马）和一条有向边（人 -[riding]-> 马），人节点可附带属性"男性""戴帽子"，马节点可附带属性"棕色""奔跑中"。

### 1.2 场景图与图像知识图谱的关系

场景图与上一章讨论的图像知识图谱既有联系又有区别：

| 对比维度 | 场景图 | 图像知识图谱 |
|---------|-------|------------|
| 范围 | 单幅图像内部的语义结构 | 跨图像、跨资源的知识网络 |
| 节点来源 | 仅限当前图像中的物体 | 可包含概念、场景等抽象实体 |
| 关系类型 | 以视觉关系为主 | 视觉关系 + 语义关系 + 跨图关系 |
| 用途 | 图像内容理解与检索 | 知识推理与知识发现 |
| 生成方式 | 场景图生成模型自动产出 | 多源融合构建 |

简言之，场景图是图像知识图谱的"基本单元"——每一幅图像贡献一个场景图，众多场景图汇聚成图像知识图谱。在 Neo4j 中，每幅图像的场景图可以作为以图像节点为根的子图存在。

### 1.3 场景图的形式化定义

形式化地，一幅图像的场景图可定义为三元组 $G = (O, R, A)$，其中：

- $O = \{o_1, o_2, ..., o_n\}$ 为物体集合，每个物体 $o_i$ 包含类别标签与边界框；
- $R = \{(s_i, p_i, t_i)\}$ 为关系三元组集合，$s_i$ 为主体（Subject），$p_i$ 为谓词（Predicate），$t_i$ 为客体（Object），表示"$s_i$ 对 $t_i$ 做了 $p_i$"；
- $A = \{(o_i, a_j)\}$ 为属性集合，表示物体 $o_i$ 具有属性 $a_j$。

## 二、场景图生成任务

### 2.1 任务定义

场景图生成（Scene Graph Generation，SGG）是指给定一幅图像，自动生成其对应场景图的任务。这是一个典型的结构化预测（Structured Prediction）问题：模型不仅要识别图像中有哪些物体，还要判断物体两两之间的关系，最终输出一个完整的图结构。

SGG 是连接计算机视觉（Computer Vision，CV）与知识图谱（Knowledge Graph，KG）的桥梁。它的输出天然是图结构化的三元组，可以直接作为知识图谱的边写入 Neo4j。

### 2.2 SGG 的三个子任务

根据预测条件的不同，场景图生成可分解为三个层次的子任务：

| 子任务 | 英文名称 | 已知条件 | 预测目标 | 难度 |
|-------|---------|---------|---------|------|
| 谓词分类 | Predicate Classification | 物体标签 + 边界框 | 物体对的谓词 | 低 |
| 场景图分类 | Scene Graph Classification | 边界框（标签未知） | 物体标签 + 谓词 | 中 |
| 场景图生成 | Scene Graph Generation | 仅图像 | 边界框 + 标签 + 谓词 | 高 |

```mermaid
flowchart LR
    A[原始图像] --> B[物体检测<br/>输出边界框]
    B --> C[物体分类<br/>赋予类别标签]
    C --> D[谓词分类<br/>判断物体对关系]
    D --> E[场景图]
    style A fill:#fce4ec,stroke:#c62828
    style E fill:#e8f5e9,stroke:#388e3c
```

**物体检测（Object Detection）** 是 SGG 的第一步，负责定位图像中所有物体并输出边界框。常用模型包括 Faster R-CNN、DETR 等。

**谓词分类（Predicate Classification）** 是 SGG 的核心，对每一对物体判断它们之间的关系。由于物体对的数量随物体数呈二次增长（$n$ 个物体产生 $n(n-1)$ 个物体对），谓词分类的计算开销较大。

**关系分类（Relationship Classification）** 在谓词分类基础上增加了对"关系是否存在"的判断——并非所有物体对都有关系，模型需要区分"有关系"与"无关系"。

## 三、视觉关系检测

### 3.1 视觉关系的形式

视觉关系（Visual Relationship）是场景图的基本构成单元，采用"主体-谓词-客体"（Subject-Predicate-Object）三元组形式表示。例如：

- `(man, riding, horse)`：人骑马
- `(cup, on, table)`：杯子在桌上
- `(girl, holding, book)`：女孩拿着书

每条视觉关系可形式化为 $(s, p, o)$，其中 $s$ 与 $o$ 是物体（含类别与位置），$p$ 是关系谓词。场景图即为所有视觉关系的集合。

### 3.2 视觉关系检测的挑战

视觉关系检测（Visual Relationship Detection，VRD）面临三大挑战：

1. **组合爆炸**：物体类别与谓词类别的笛卡尔积极大，许多组合在训练集中从未出现，导致零样本（Zero-shot）关系检测困难；
2. **视觉信号弱**：关系往往无法从局部视觉特征直接判断，如"骑"与"坐"在视觉上极其相似，需要全局上下文；
3. **长尾分布**：少数高频关系（如 on、has、wearing）占绝大多数样本，大量低频关系样本稀少。

> **图书情报视角**：视觉关系的长尾分布问题与文献计量学中的 Bradford 定律（文献分散规律）和 Zipf 定律（词频分布规律）高度相似——少数核心关系类型覆盖大量样本，大量边缘关系类型各占少量样本。这启示我们可借鉴文献计量学的方法，对关系类型进行核心区与边缘区的划分，针对性地设计检索策略。

## 四、经典 SGG 模型对比

### 4.1 模型演进脉络

场景图生成模型经历了从"先检测后配对"到"端到端联合建模"再到"去偏公平建模"的演进。下表对比四种经典模型：

| 模型 | 全称 | 核心思想 | 关系建模方式 | 优势 | 局限 |
|------|------|---------|------------|------|------|
| ImpDETR | Improved DETR | 基于 DETR 检测器，将关系作为"关系三元组"直接检测 | 端到端集合预测 | 端到端、可并行 | 训练收敛慢 |
| MOTIFS | Motif-based SGG | 利用视觉关系中的motif（模式）统计规律，逐阶段建模 | 两阶段：物体→关系 | 建模上下文依赖 | 计算开销较大 |
| VCTree | Visual Context Tree | 用动态二叉树组织物体间的上下文，自适应选择遍历顺序 | 树结构上下文 | 灵活适应不同场景 | 实现复杂 |
| Unbiased SGG | Unbiased Scene Graph Generation | 通过重采样与因果推断消除谓词分布偏置 | 因果去偏 | 公平性提升 | 高频关系召回下降 |

### 4.2 模型详细解析

**MOTIFS** 模型的核心洞见是：图像中的物体分布与关系分布存在显著的motif（模式）规律。例如，"人-骑-马"这一三元组远比"马-骑-人"常见，这种统计模式可用于约束关系预测。MOTIFS 采用两阶段建模：第一阶段预测物体标签，第二阶段在已知物体标签的条件下预测关系。

**VCTree** 模型认为，固定的图遍历顺序（如从左到右）不总是最优的。它构建一棵动态的视觉上下文树（Visual Context Tree），根据图像内容自适应地决定哪些物体应该被优先关联，从而更灵活地捕获长程依赖。

**Unbiased SGG** 模型关注的是公平性问题。传统 SGG 模型在训练数据上存在严重的谓词分布偏置——模型倾向于预测高频谓词（如 on、has），而忽视低频但有意义的谓词（如 painted on、parked on）。Unbiased SGG 通过因果推断中的反事实推理（Counterfactual Reasoning），剥离训练数据带来的偏置，使模型对各类谓词做出更公平的预测。

### 4.3 评价指标

SGG 模型通常使用 Recall@K（R@K）作为评价指标，即在返回的前 K 个结果中包含正确关系的比例。由于场景图生成中正样本远少于负样本，使用 Precision 会导致虚高，因此采用 Recall。

| 指标 | 含义 | 适用场景 |
|------|------|---------|
| R@50 | 前 50 个预测中正确关系的比例 | 评估召回能力 |
| R@100 | 前 100 个预测中正确关系的比例 | 评估召回能力 |
| mR@50 | 各谓词类别 R@50 的平均值 | 评估公平性（关注低频关系） |
| F1@50 | Precision 与 Recall 的调和平均 | 综合评估 |

## 五、场景图与 Neo4j 知识图谱的对接

### 5.1 从场景图到 Neo4j 子图

场景图生成模型的输出可以直接映射为 Neo4j 中的图结构。下图展示了这一对接过程：

```mermaid
flowchart LR
    A[图像] --> B[SGG 模型]
    B --> C[场景图<br/>三元组集合]
    C --> D[图结构映射]
    D --> E[Neo4j 子图]
    style A fill:#fce4ec,stroke:#c62828
    style E fill:#e8f5e9,stroke:#388e3c
```

映射规则如下：

- 场景图中的每个物体 → Neo4j 中的 `Object` 节点；
- 场景图中的每条关系三元组 → Neo4j 中的一条有向边；
- 场景图中的物体属性 → Neo4j 节点的属性。

### 5.2 写入示例

假设 SGG 模型对一幅图像输出了如下三元组：

```
(person, wearing, hat)
(person, riding, horse)
(horse, on, grass)
```

对应的 Cypher 写入语句为：

```cypher
// 创建物体节点
CREATE (p:Object {object_id: 'obj_p1', label: 'person', bbox: [120, 80, 340, 520]})
CREATE (h:Object {object_id: 'obj_h1', label: 'hat', bbox: [180, 60, 250, 130]})
CREATE (horse:Object {object_id: 'obj_h2', label: 'horse', bbox: [200, 200, 600, 550]})
CREATE (g:Object {object_id: 'obj_g1', label: 'grass', bbox: [0, 400, 800, 600]})

// 创建关系
CREATE (p)-[:WEARING {confidence: 0.92}]->(h)
CREATE (p)-[:RIDING {confidence: 0.88}]->(horse)
CREATE (horse)-[:ON {confidence: 0.95}]->(g);
```

### 5.3 批量场景图导入

对于大规模场景图数据（如 Visual Genome 数据集的转换），可以使用批量导入脚本：

```cypher
// 批量导入物体节点
UNWIND $objects AS obj
MERGE (o:Object {object_id: obj.object_id})
SET o.label = obj.label,
    o.bbox = obj.bbox,
    o.confidence = obj.confidence;

// 批量导入关系
UNWIND $relations AS rel
MATCH (s:Object {object_id: rel.subject_id})
MATCH (o:Object {object_id: rel.object_id})
CALL apoc.merge.relationship(s, rel.predicate,
  {confidence: rel.confidence}, {}, {}, {}) YIELD r
RETURN count(r) AS created;
```

`apoc.merge.relationship` 用于动态创建关系类型，因为谓词名称来自模型输出，无法在 Cypher 中硬编码。`apoc.merge.relationship` 在关系已存在时会合并属性而非重复创建。

## 六、场景图在图像检索中的应用

### 6.1 基于场景图的图像检索

基于场景图的图像检索（Scene Graph based Image Retrieval，SGIR）是场景图生成最直接的应用。其基本思想是：用户用一幅"查询场景图"描述想要检索的图像内容，系统在数据库中找到场景图与之匹配的图像。

例如，用户查询"一个人骑着棕色的马"，系统将其转化为场景图 `(person)-[riding]->(horse{color:brown})`，然后在图像数据库中匹配具有相同结构的场景图。

### 6.2 检索模式

场景图支持三种检索模式：

| 检索模式 | 查询形式 | 匹配方式 | 示例 |
|---------|---------|---------|------|
| 精确匹配 | 完整场景图 | 节点与边完全一致 | 找到所有"人骑马"的图像 |
| 子图匹配 | 部分场景图 | 查询图作为子图匹配 | 找到所有包含"人骑马"的图像（无论还有其他物体） |
| 模糊匹配 | 带相似度的场景图 | 允许部分节点/边不同 | 找到"人骑马或人骑自行车"的图像 |

### 6.3 Neo4j 中的场景图检索查询

在 Neo4j 中，子图匹配检索可以直接用 Cypher 模式匹配实现：

```cypher
// 检索"人骑马"的图像
MATCH (p:Object {label: 'person'})
      -[:RIDING]->(h:Object {label: 'horse'})
MATCH (img:Image)-[:DEPICTS]->(p)
RETURN DISTINCT img.image_id, img.file_path
LIMIT 20;
```

对于更复杂的组合查询（"在草地上、人骑着马"），只需扩展模式：

```cypher
MATCH (p:Object {label: 'person'})
      -[:RIDING]->(horse:Object {label: 'horse'})
MATCH (horse)-[:ON]->(g:Object {label: 'grass'})
MATCH (img:Image)-[:DEPICTS]->(p)
RETURN DISTINCT img.image_id, img.file_path
LIMIT 20;
```

这种基于模式匹配的检索方式，使得用户可以用自然语言描述场景，系统将其转化为场景图查询，再在 Neo4j 中执行图匹配。这与图书情报领域"受控词汇检索"的理念相通——将自然语言查询映射为结构化的查询图，再与数据库中的图结构进行匹配。

## 七、小结

场景图生成是图像内容理解的核心任务，它将图像从像素空间映射到语义图空间，为图像知识图谱的构建提供了自动化手段。SGG 的三个子任务（物体检测、谓词分类、关系分类）层层递进，经典模型从 ImpDETR 到 Unbiased SGG 不断演进，在准确性与公平性上持续提升。

场景图的输出天然是图结构化的三元组，可以无缝对接 Neo4j 知识图谱。在 Neo4j 中，场景图成为以图像为根的子图，支持基于 Cypher 模式匹配的细粒度图像检索。下一章将深入探讨场景图的基本构成单元——视觉关系检测的细节。

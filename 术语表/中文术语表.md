# 中文术语表

> 本术语表按主题分类整理知识库涉及的核心术语，每个术语包含中文译名与简要定义。
> **使用说明**：术语按五大主题分类组织，同一术语可能出现在多个类别中。如需按字母检索，请参阅[中英对照术语表](./glossary-en-zh.md)。

---

## 一、图论基础

| 序号 | 中文术语 | 英文术语 | 简要定义 |
|------|---------|---------|---------|
| 1 | 图 | Graph | 由节点（顶点）集合与边（关系）集合构成的数学结构，用于表示实体间的拓扑关系 |
| 2 | 节点 | Node / Vertex | 图中的基本元素，表示一个实体对象，也称为顶点 |
| 3 | 边 | Edge / Link | 图中连接两个节点的元素，表示节点间的关系，也称为链接 |
| 4 | 有向图 | Directed Graph | 边具有方向性的图，每条边从源节点指向目标节点 |
| 5 | 无向图 | Undirected Graph | 边没有方向性的图，节点间的关系是对称的 |
| 6 | 权重 | Weight | 附着在边上的数值，表示关系的强度、距离或成本 |
| 7 | 度 | Degree | 与某节点直接相连的边的数量；有向图中分为入度与出度 |
| 8 | 入度 | In-degree | 有向图中指向某节点的边的数量 |
| 9 | 出度 | Out-degree | 有向图中从某节点出发的边的数量 |
| 10 | 路径 | Path | 图中由依次相连的边组成的序列，连接起始节点与终止节点 |
| 11 | 最短路径 | Shortest Path | 图中两个节点之间边数最少或权重之和最小的路径 |
| 12 | 连通图 | Connected Graph | 任意两个节点之间都存在路径的无向图 |
| 13 | 连通分量 | Connected Component | 图中最大的连通子图，分量内任意两节点可达，分量间不可达 |
| 14 | 子图 | Subgraph | 由原图的部分节点与部分边构成的图 |
| 15 | 邻居 | Neighbor | 与某节点直接通过边相连的其他节点 |
| 16 | 邻域 | Neighborhood | 某节点的所有邻居节点的集合 |
| 17 | 环 | Cycle | 起点与终点为同一节点的路径，即闭合路径 |
| 18 | 二分图 | Bipartite Graph | 节点可分为两个不相交的子集，且每条边连接分属不同子集的节点的图 |
| 19 | 多重图 | Multigraph | 允许两个节点之间存在多条边的图 |
| 20 | 非欧几里得数据 | Non-Euclidean Data | 不具备规则网格结构（如平移不变性）的数据，图数据是其典型代表 |

## 二、Neo4j 与 Cypher

| 序号 | 中文术语 | 英文术语 | 简要定义 |
|------|---------|---------|---------|
| 21 | 属性图模型 | Property Graph Model | 以节点、关系、属性、标签为核心要素的图数据模型，Neo4j 采用的数据模型 |
| 22 | 标签 | Label | 附加在节点上的类型标记，用于对节点分类，一个节点可拥有多个标签 |
| 23 | 关系类型 | Relationship Type | 对边的分类标记，每条关系必须有一个且仅有一个类型 |
| 24 | 属性 | Property | 以键值对形式存储在节点或关系上的数据字段 |
| 25 | 无索引邻接 | Index-Free Adjacency | Neo4j 的核心存储技术，每个节点直接存储其邻居的物理指针，使图遍历复杂度与图规模无关 |
| 26 | Cypher | Cypher | Neo4j 的声明式图查询语言，采用模式匹配语法表达图查询 |
| 27 | 模式匹配 | Pattern Matching | Cypher 的核心机制，通过 `MATCH` 子句描述图的拓扑模式以检索匹配的子图 |
| 28 | 节点模式 | Node Pattern | Cypher 中用括号表示的节点匹配表达式，如 `(n:Person {name: "Alice"})` |
| 29 | 关系模式 | Relationship Pattern | Cypher 中用箭头与方括号表示的关系匹配表达式，如 `-[r:KNOWS]->` |
| 30 | 变长路径 | Variable-Length Path | Cypher 中使用 `*` 指定可变跳数的路径模式，如 `-[*1..3]->` 表示1到3跳 |
| 31 | 约束 | Constraint | 保证数据完整性的规则，如唯一性约束确保某属性值不重复 |
| 32 | 索引 | Index | 为加速查询而建立的辅助数据结构，Neo4j 支持 B 树索引与向量索引 |
| 33 | APOC | APOC | Awesome Procedures On Cypher，Neo4j 社区维护的实用过程与函数库 |
| 34 | Neo4j Browser | Neo4j Browser | Neo4j 内置的 Web 图查询与可视化工具 |
| 35 | Neo4j Aura | Neo4j Aura | Neo4j 的全托管云端图数据库服务 |
| 36 | Bolt 协议 | Bolt Protocol | Neo4j 设计的二进制网络协议，用于客户端与数据库之间的高效通信 |
| 37 | 事务 | Transaction | 保证 ACID 特性的数据库操作单元，Neo4j 支持完全 ACID 事务 |

## 三、GDS 图数据科学

| 序号 | 中文术语 | 英文术语 | 简要定义 |
|------|---------|---------|---------|
| 38 | 图数据科学 | Graph Data Science (GDS) | 运用图算法与图机器学习从图结构数据中提取知识与洞察的学科领域，也是 Neo4j 算法库的名称 |
| 39 | 图投影 | Graph Projection | GDS 将数据库中的图数据加载到内存中专用图结构的过程，是运行算法的前置步骤 |
| 40 | 图目录 | Graph Catalog | GDS 中管理图投影的组件，负责图的创建、删除、导出与统计 |
| 41 | 中心性 | Centrality | 衡量节点在图中重要性的指标族，包括度中心性、接近中心性、介数中心性、PageRank 等 |
| 42 | PageRank | PageRank | 基于随机游走模型的节点重要性算法，通过邻居的重要性迭代计算当前节点的重要性得分 |
| 43 | 介数中心性 | Betweenness Centrality | 衡量节点出现在其他节点对最短路径上的频率，反映节点作为"桥梁"的程度 |
| 44 | 接近中心性 | Closeness Centrality | 衡量节点到图中所有其他节点的平均最短距离的倒数，反映节点的全局可达性 |
| 45 | 度中心性 | Degree Centrality | 以节点的度（连接数）直接作为重要性度量，是最简单的中心性指标 |
| 46 | 社区发现 | Community Detection | 识别图中紧密连接的节点群组的算法族，如 Louvain、标签传播、连通分量 |
| 47 | Louvain 算法 | Louvain Algorithm | 基于模块度优化的层次化社区发现算法，通过局部节点移动与图凝聚实现多尺度社区划分 |
| 48 | 标签传播 | Label Propagation | 通过邻居标签投票迭代传播社区归属的快速社区发现算法 |
| 49 | 连通分量 | Connected Components | 识别图中所有连通子图的算法，每个连通分量是一个独立的节点群组 |
| 50 | 三角计数 | Triangle Counting | 统计每个节点参与的三角形（三个节点两两相连）数量的算法，用于衡量局部聚类程度 |
| 51 | 模块度 | Modularity | 衡量社区划分质量的指标，比较社区内实际边数与随机情况下的期望边数 |
| 52 | 相似度算法 | Similarity Algorithm | 计算节点或向量之间相似程度的算法族，包括 Jaccard、余弦相似度、Pearson 等 |
| 53 | Jaccard 相似度 | Jaccard Similarity | 两个集合交集大小与并集大小之比，取值范围 [0, 1] |
| 54 | 余弦相似度 | Cosine Similarity | 两个向量夹角的余弦值，衡量向量方向的相似性，忽略向量长度 |
| 55 | 路径算法 | Pathfinding Algorithm | 在图中寻找满足特定条件的路径的算法族，如 Dijkstra、A*、Yen's 算法 |
| 56 | Dijkstra 算法 | Dijkstra's Algorithm | 在非负权重图中寻找单源最短路径的经典算法 |
| 57 | A*算法 | A* Algorithm | 结合启发式估计的最短路径算法，通过实际代价与估计代价之和引导搜索方向 |
| 58 | 全对最短路径 | All Pairs Shortest Path | 计算图中所有节点对之间最短路径的算法 |

## 四、图神经网络与嵌入

| 序号 | 中文术语 | 英文术语 | 简要定义 |
|------|---------|---------|---------|
| 59 | 图嵌入 | Graph Embedding | 将图结构数据（节点、边、子图）映射到低维向量空间的技术，是连接图与机器学习的桥梁 |
| 60 | 节点嵌入 | Node Embedding | 为图中每个节点生成低维向量表示，使图中相似的节点在向量空间中距离相近 |
| 61 | 图神经网络 | Graph Neural Network (GNN) | 直接在图结构数据上运行的神经网络，通过邻居信息聚合学习节点或图的表示 |
| 62 | 消息传递 | Message Passing | GNN 的核心机制，节点通过从邻居接收"消息"（变换后的特征）并聚合来更新自身表示 |
| 63 | 谱方法 | Spectral Method | 基于图拉普拉斯矩阵的谱分解定义图卷积的方法，GCN 是其一阶近似特例 |
| 64 | 空间方法 | Spatial Method | 直接在节点邻域上定义卷积操作的方法，如 GraphSAGE、GAT，不依赖谱分解 |
| 65 | 图卷积网络 | Graph Convolutional Network (GCN) | Kipf & Welling (2017) 提出的图神经网络，通过一阶谱近似的逐层传播规则实现高效的图卷积 |
| 66 | 图注意力网络 | Graph Attention Network (GAT) | 引入注意力机制的图神经网络，自适应地为不同邻居分配不同权重 |
| 67 | 图拉普拉斯 | Graph Laplacian | 图的拉普拉斯矩阵（度矩阵减邻接矩阵），其特征值与特征向量揭示图的谱性质 |
| 68 | 随机游走 | Random Walk | 在图上按一定概率规则从一个节点游走到其邻居的随机过程，是 node2vec、DeepWalk 的基础 |
| 69 | node2vec | node2vec | 通过二阶随机游走学习节点嵌入的算法，通过参数 p 和 q 平衡广度优先与深度优先搜索策略 |
| 70 | DeepWalk | DeepWalk | 最早的基于随机游走的节点嵌入方法，将随机游走序列类比为句子，用 Word2vec 训练嵌入 |
| 71 | FastRP | FastRP (Fast Random Projection) | 基于随机投影的高效图嵌入算法，无需迭代，时间复杂度近似线性，适用于大规模图 |
| 72 | GraphSAGE | GraphSAGE | 归纳式图神经网络，通过学习可复用的邻居聚合函数为未见节点生成嵌入 |
| 73 | 直推式学习 | Transductive Learning | 在训练时需要见到全部节点（含测试节点）的学习范式，如 GCN、DeepWalk |
| 74 | 归纳式学习 | Inductive Learning | 在训练时仅依赖训练节点，能够泛化到未见节点的学习范式，如 GraphSAGE |
| 75 | 聚合函数 | Aggregator Function | GraphSAGE 中将邻居特征聚合为单一向量的函数，包括均值、LSTM、最大池化等 |
| 76 | 多头注意力 | Multi-Head Attention | GAT 中使用多组独立的注意力机制并行计算，拼接或平均以增强表达能力的技术 |
| 77 | 同质性 | Homophily | 图中相似节点倾向于相连的性质，即"物以类聚"现象 |
| 78 | 结构等价性 | Structural Equivalence | 节点在图结构中扮演相似角色（拥有相似的邻域拓扑）的性质，即使不相邻也视为相似 |
| 79 | 图采样 | Graph Sampling | 从大图中抽取子集用于训练的技术，是 GNN 可扩展性的关键，如邻居采样、层采样 |
| 80 | 知识图谱嵌入 | Knowledge Graph Embedding | 将知识图谱中的实体与关系映射到向量空间的技术，如 TransE、TransR、RotatE |

## 五、AI 图像应用

| 序号 | 中文术语 | 英文术语 | 简要定义 |
|------|---------|---------|---------|
| 81 | 图像知识图谱 | Image Knowledge Graph | 以图像中的视觉实体为节点、视觉关系为边构建的知识图谱，实现图像的结构化表示 |
| 82 | 场景图 | Scene Graph | 描述图像场景的有向图，节点为物体（含属性），边为物体间的关系，是图像知识图谱的一种形式 |
| 83 | 场景图生成 | Scene Graph Generation (SGG) | 从图像中自动生成场景图的任务，涉及物体检测、关系预测与图结构构建 |
| 84 | 视觉关系检测 | Visual Relationship Detection | 识别图像中两个物体之间语义关系（如"人骑马""书在桌上"）的任务 |
| 85 | 物体检测 | Object Detection | 在图像中定位并识别物体的任务，输出物体的边界框与类别标签 |
| 86 | 边界框 | Bounding Box | 用矩形框标注图像中物体位置与范围的坐标表示 |
| 87 | 视觉基因组 | Visual Genome | 大规模图像场景图标注数据集，包含物体、属性、关系的密集标注，是场景图生成研究的基准数据集 |
| 88 | 子图匹配 | Subgraph Matching | 在大图中查找与给定查询图结构相同的子图的操作，用于场景图检索 |
| 89 | 图检索增强生成 | Graph RAG (GraphRAG) | 结合知识图谱与大语言模型的检索增强生成方法，通过图结构实现结构化检索与推理 |
| 90 | 检索增强生成 | Retrieval-Augmented Generation (RAG) | 在生成式模型推理时从外部知识库检索相关信息以增强回答质量的技术范式 |
| 91 | 向量索引 | Vector Index | 为加速向量相似度搜索而建立的索引结构，如 HNSW、IVF，Neo4j 5.x 起原生支持 |
| 92 | 向量嵌入 | Vector Embedding | 将文本、图像等数据映射为高维浮点向量的表示，语义相似的数据在向量空间中距离相近 |
| 93 | 大语言模型 | Large Language Model (LLM) | 基于Transformer架构在海量文本上预训练的超大规模语言模型，如 GPT、LLaMA |
| 94 | 混合检索 | Hybrid Retrieval | 结合多种检索策略（如向量语义检索与图结构遍历）的检索方法，GraphRAG 的核心特征 |
| 95 | 图原生 AI | Graph-Native AI | 以图结构为核心数据表示的 AI 范式，将知识图谱与关系推理能力深度融入 AI 模型 |
| 96 | 视觉实体 | Visual Entity | 图像中被识别出的具体物体或概念，是图像知识图谱的节点 |
| 97 | 视觉关系 | Visual Relationship | 图像中两个视觉实体之间的语义关联，如空间关系（"在...上"）、动作关系（"骑"）等 |
| 98 | 图像检索 | Image Retrieval | 根据查询条件（文本、图像、场景图等）从图像数据库中查找相关图像的任务 |

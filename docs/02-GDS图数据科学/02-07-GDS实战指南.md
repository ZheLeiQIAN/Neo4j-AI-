## 二、数据准备

### 2.1 导入图布局样本数据

我们使用一个简化的图像知识图谱数据集，包含图像、物体、类别及其关系。这一数据集对应本知识库 [Cypher 示例](../01-foundations/01-05-cypher-examples.md) 中的模型，但规模稍大，适合演示算法效果。

```cypher
// 清空数据库（实验环境）
MATCH (n) DETACH DELETE n;

// 第一步：创建约束
CREATE CONSTRAINT FOR (img:Image) REQUIRE img.filename IS UNIQUE;
CREATE CONSTRAINT FOR (o:Object) REQUIRE o.object_id IS UNIQUE;
CREATE CONSTRAINT FOR (c:Category) REQUIRE c.name IS UNIQUE;

// 第二步：批量创建类别
UNWIND [
    {name: "person", super: "living_being"},
    {name: "phone", super: "electronics"},
    {name: "chair", super: "furniture"},
    {name: "laptop", super: "electronics"},
    {name: "book", super: "object"},
    {name: "table", super: "furniture"},
    {name: "car", super: "vehicle"},
    {name: "dog", super: "living_being"}
] AS row
MERGE (c:Category {name: row.name})
SET c.super_category = row.super;

// 第三步：批量创建图像
UNWIND [
    {fn: "img001.jpg", w: 1920, h: 1080},
    {fn: "img002.jpg", w: 1280, h: 720},
    {fn: "img003.jpg", w: 1920, h: 1080},
    {fn: "img004.jpg", w: 1080, h: 1920},
    {fn: "img005.jpg", w: 1280, h: 720},
    {fn: "img006.jpg", w: 1920, h: 1080}
] AS row
MERGE (img:Image {filename: row.fn})
SET img.width = row.w, img.height = row.h;

// 第四步：批量创建物体和检测关系
UNWIND [
    {oid: "o001", cat: "person", conf: 0.95, img: "img001.jpg"},
    {oid: "o002", cat: "phone", conf: 0.88, img: "img001.jpg"},
    {oid: "o003", cat: "chair", conf: 0.92, img: "img001.jpg"},
    {oid: "o004", cat: "laptop", conf: 0.90, img: "img002.jpg"},
    {oid: "o005", cat: "person", conf: 0.87, img: "img002.jpg"},
    {oid: "o006", cat: "book", conf: 0.85, img: "img002.jpg"},
    {oid: "o007", cat: "person", conf: 0.93, img: "img003.jpg"},
    {oid: "o008", cat: "chair", conf: 0.89, img: "img003.jpg"},
    {oid: "o009", cat: "car", conf: 0.96, img: "img004.jpg"},
    {oid: "o010", cat: "person", conf: 0.91, img: "img004.jpg"},
    {oid: "o011", cat: "dog", conf: 0.94, img: "img005.jpg"},
    {oid: "o012", cat: "person", conf: 0.88, img: "img005.jpg"},
    {oid: "o013", cat: "laptop", conf: 0.86, img: "img006.jpg"},
    {oid: "o014", cat: "table", conf: 0.91, img: "img006.jpg"}
] AS row
MERGE (o:Object {object_id: row.oid})
SET o.category = row.cat, o.confidence = row.conf
WITH o, row
MATCH (img:Image {filename: row.img})
MERGE (img)-[:DETECTS]->(o)
WITH o
MATCH (c:Category {name: o.category})
MERGE (o)-[:IS_A]->(c);

// 第五步：创建物体间语义关系
UNWIND [
    {s: "o001", t: "o002", rel: "HOLDING", conf: 0.85},
    {s: "o001", t: "o003", rel: "SITTING_ON", conf: 0.90},
    {s: "o005", t: "o004", rel: "USING", conf: 0.82},
    {s: "o005", t: "o006", rel: "READING", conf: 0.78},
    {s: "o007", t: "o008", rel: "SITTING_ON", conf: 0.88},
    {s: "o010", t: "o009", rel: "NEAR", conf: 0.75},
    {s: "o012", t: "o011", rel: "PETTING", conf: 0.80}
] AS row
MATCH (s:Object {object_id: row.s}), (t:Object {object_id: row.t})
CALL apoc.merge.relationship(s, row.rel, {confidence: row.conf}, {}, t, {}) YIELD rel
RETURN count(rel) AS createdRelations;

// 第六步：创建图像间相似关系
UNWIND [
    {s: "img001.jpg", t: "img003.jpg", score: 0.85},
    {s: "img002.jpg", t: "img006.jpg", score: 0.78},
    {s: "img001.jpg", t: "img002.jpg", score: 0.65},
    {s: "img004.jpg", t: "img005.jpg", score: 0.60}
] AS row
MATCH (s:Image {filename: row.s}), (t:Image {filename: row.t})
MERGE (s)-[:SIMILAR_TO {score: row.score}]->(t);
```

## 三、完整工作流演示

下面演示 GDS 的标准五步工作流：创建图投影 → 运行中心性算法 → 运行社区发现 → 计算节点相似性 → 结果写回与可视化。

### 3.1 第一步：创建图投影

```cypher
// 创建图像相似网络图投影（用于中心性和社区发现）
CALL gds.graph.project(
    'imageSimGraph',
    {
        Image: {
            properties: ['filename']
        }
    },
    {
        SIMILAR_TO: {
            orientation: 'UNDIRECTED',
            properties: ['score']
        }
    }
)
YIELD graphName, nodeCount, relationshipCount, projectMillis
RETURN graphName, nodeCount, relationshipCount, projectMillis;
```

```cypher
// 创建图像-物体二部图投影（用于相似性计算）
CALL gds.graph.project(
    'imageObjectGraph',
    ['Image', 'Object'],
    {
        DETECTS: {orientation: 'UNDIRECTED'}
    }
)
YIELD graphName, nodeCount, relationshipCount;
```

### 3.2 第二步：运行中心性算法

```cypher
// 在图像相似网络上运行 PageRank，识别代表性图像
CALL gds.pageRank.write('imageSimGraph', {
    writeProperty: 'imageImportance',
    relationshipWeightProperty: 'score',
    dampingFactor: 0.85,
    maxIterations: 100
})
YIELD nodePropertiesWritten, ranIterations;

// 查看结果
MATCH (img:Image)
RETURN img.filename AS image, img.imageImportance AS importance
ORDER BY importance DESC;

// 清理图投影
CALL gds.graph.drop('imageSimGraph');
```

### 3.3 第三步：运行社区发现

```cypher
// 重新创建图投影（上一步已删除）
CALL gds.graph.project(
    'imageCommunity',
    'Image',
    {
        SIMILAR_TO: {
            orientation: 'UNDIRECTED',
            properties: ['score']
        }
    }
);

// 运行 Louvain 社区发现，对图像聚类
CALL gds.louvain.write('imageCommunity', {
    writeProperty: 'imageCluster',
    relationshipWeightProperty: 'score'
})
YIELD communityCount, modularity, ranLevels;

// 查看聚类结果
MATCH (img:Image)
RETURN img.imageCluster AS cluster,
       count(img) AS imageCount,
       collect(img.filename) AS images
ORDER BY imageCount DESC;

CALL gds.graph.drop('imageCommunity');
```

### 3.4 第四步：计算节点相似性

```cypher
// 在二部图上计算图像间的结构相似性（基于共同物体）
CALL gds.nodeSimilarity.write('imageObjectGraph', {
    writeRelationshipType: 'CONTENT_SIMILAR',
    writeProperty: 'structSim',
    similarityMetric: 'JACCARD',
    similarityCutoff: 0.1,
    topK: 3
})
YIELD nodesCompared, relationshipsWritten;

// 查看内容相似的图像对
MATCH (img1:Image)-[r:CONTENT_SIMILAR]->(img2:Image)
RETURN img1.filename, img2.filename, r.structSim
ORDER BY r.structSim DESC;

CALL gds.graph.drop('imageObjectGraph');
```

### 3.5 第五步：结果写回与可视化

经过前四步，分析结果已写回数据库节点属性。现在可以用 Cypher 查询进行综合分析和可视化。

```cypher
// 综合查看每幅图像的分析结果
MATCH (img:Image)
RETURN img.filename AS image,
       img.imageImportance AS importance,
       img.imageCluster AS cluster,
       [(img)-[r:CONTENT_SIMILAR]->(i2:Image) | i2.filename] AS similarImages
ORDER BY importance DESC;
```

在 Neo4j Browser 中可视化：

```cypher
// 可视化图像相似网络（带社区颜色）
MATCH (img:Image)-[r:SIMILAR_TO|CONTENT_SIMILAR*1..1]-(img2:Image)
RETURN img, r, img2
LIMIT 50;
```

## 四、Python 客户端操作示例

以下是使用 Python 客户端 `graphdatascience` 完成同样工作流的代码，适合与数据分析生态集成。

```python
from graphdatascience import GraphDataScience
import pandas as pd

# 连接 Neo4j
gds = GraphDataScience("bolt://localhost:7687", auth=("neo4j", "password"))

# 第一步：创建图投影
G, result = gds.graph.project(
    "image_analysis",
    "Image",
    {"SIMILAR_TO": {"orientation": "UNDIRECTED", "properties": ["score"]}}
)
print(f"图投影创建: {result}")

# 第二步：运行 PageRank
pr_df = gds.pageRank.stream(
    G,
    relationshipWeightProperty="score"
)
pr_df["filename"] = pr_df["nodeId"].apply(
    lambda id: gds.util.asNode(id)["filename"]
)
print("\n图像重要性排名:")
print(pr_df[["filename", "score"]].sort_values("score", ascending=False))

# 第三步：运行 Louvain 社区发现
gds.louvain.write(
    G,
    writeProperty="imageCluster",
    relationshipWeightProperty="score"
)

# 第四步：查询结果并可视化
result_df = gds.run_cypher("""
    MATCH (img:Image)
    RETURN img.filename AS image,
           img.imageImportance AS importance,
           img.imageCluster AS cluster
    ORDER BY importance DESC
""")
print("\n综合分析结果:")
print(result_df)

# 第五步：用 matplotlib 可视化聚类
import matplotlib.pyplot as plt

clusters = result_df.groupby("cluster")["image"].count()
plt.figure(figsize=(8, 5))
plt.bar(clusters.index, clusters.values)
plt.xlabel("Cluster ID")
plt.ylabel("Image Count")
plt.title("Image Clustering by Louvain")
plt.savefig("image_clusters.png")
print("可视化已保存为 image_clusters.png")

# 清理
G.drop()
print("图投影已删除")
```

## 五、常见问题与调优建议

### 5.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 内存不足（OOM） | 图投影过大 | 增大堆内存或缩小投影范围 |
| 算法不收敛 | 参数设置不当 | 增大 `maxIterations` 或调整 `tolerance` |
| 结果为空 | 图投影为空 | 检查标签和关系类型是否正确 |
| 性能慢 | 缺少索引 | 为查询字段创建索引 |
| 权重未生效 | 属性未加载 | 在投影配置中指定 `properties` |

### 5.2 性能调优建议

| 调优方向 | 建议 | 说明 |
|---------|------|------|
| 内存配置 | 堆内存设为可用内存的 60-70% | 避免频繁 GC |
| 图投影 | 使用原生投影而非 Cypher 投影 | 性能差距可达 10 倍 |
| 算法选择 | 大图用近似算法 | 如 Betweenness 采样版 |
| 并发度 | 调整 `concurrency` 参数 | 利用多核 CPU |
| 图分区 | 大图按时间/类别分区 | 分批分析降低内存压力 |
| 索引优化 | 为 `write` 模式创建属性索引 | 加速写回 |

### 5.3 最佳实践清单

```cypher
// 1. 创建图投影前先估算内存
CALL gds.graph.project.estimate('Paper', 'CITES')
YIELD requiredMemory, nodeCount, relationshipCount;

// 2. 用 stats 模式快速评估算法效果
CALL gds.louvain.stats('myGraph')
YIELD communityCount, modularity;

// 3. 确认效果后再用 write 模式持久化
CALL gds.louvain.write('myGraph', {writeProperty: 'community'});

// 4. 创建索引加速写回后的查询
CREATE INDEX FOR (p:Paper) ON (p.community);

// 5. 使用完毕后删除图投影
CALL gds.graph.drop('myGraph');

// 6. 定期清理所有图投影（防止内存泄漏）
CALL gds.graph.list()
YIELD graphName
WITH collect(graphName) AS names
UNWIND names AS name
CALL gds.graph.drop(name) YIELD graphName
RETURN graphName;
```

## 六、与图书情报领域的关联

本实战指南的图像知识图谱分析流程，可以直接迁移到图书情报领域的引文网络分析。下表展示了两个领域的流程对应关系。

| 工作流步骤 | 图像领域（本指南） | 图书情报领域 | 使用的算法 |
|-----------|------------------|------------|-----------|
| 数据准备 | 图像-物体-类别图谱 | 论文-作者-期刊图谱 | MERGE/CREATE |
| 创建图投影 | 图像相似网络 | 引文网络 | gds.graph.project |
| 中心性分析 | 识别代表性图像 | 评估论文影响力 | PageRank |
| 社区发现 | 图像聚类 | 研究主题发现 | Louvain |
| 相似性计算 | 内容相似图像 | 相似文献推荐 | Node Similarity |
| 结果可视化 | 图像相似网络图 | 引文聚类图 | Neo4j Browser |

这种流程对应关系体现了 GDS 工作流的通用性——同一套分析框架，只需替换数据模型和领域解读，就能服务于不同的应用场景。对于信息资源管理领域的研究者，掌握这套工作流意味着拥有了一个强大的图分析工具箱。

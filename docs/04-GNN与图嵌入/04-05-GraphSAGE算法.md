## 一、GraphSAGE 的核心动机：归纳式学习

GraphSAGE（Graph Sample and Aggregate，Hamilton 等，2017）是图神经网络发展史上的里程碑工作。它的核心动机直指 Node2Vec、DeepWalk、FastRP 等直推式方法的根本局限：**这些方法为每个节点学习一个固定的向量，无法处理训练时未曾见过的节点。**

在真实世界中，图是动态变化的：社交网络每天都有新用户注册，引文网络每天都有新论文发表，知识图谱每天都有新实体加入。对于直推式方法，每加入一个新节点，就必须把整张图重新嵌入一遍——这在工程上是不可接受的。

GraphSAGE 的革命性在于：它不学习每个节点的具体向量，而是学习一个**从节点邻域到向量的聚合函数**（Aggregation Function）。给定任意节点（哪怕是训练时未见的新节点），只要它的邻域结构已知，就能用学到的函数生成它的嵌入。这就是归纳式学习（Inductive Learning）的本质——学的是"规则"而非"实例"。

GraphSAGE 这一名字概括了它的两个核心机制：**Sample**（邻域采样）与 **Aggregate**（邻居聚合）。

## 二、邻域采样策略

### 2.1 为什么需要采样

GraphSAGE 的消息传递需要聚合每个节点的邻居信息。如果直接聚合全部邻居，会遇到两个问题：

1. **计算爆炸**：高度数节点的邻居可达数千甚至数万，全聚合的计算与内存成本极高；
2. **方差过大**：邻居数量差异巨大导致聚合结果的尺度不一致，影响训练稳定性。

GraphSAGE 引入邻域采样（Neighborhood Sampling）解决这两个问题：对每个节点，固定采样 S 个邻居（而非全部），保证计算量可控且尺度一致。

### 2.2 采样机制

GraphSAGE 采用多跳采样。设采样规模为 [S_1, S_2, ..., S_K]，其中 K 是 GNN 层数，S_k 是第 k 层的采样数。对于目标节点 v：

- 第 1 层：从 v 的邻居中随机采样 S_1 个；
- 第 2 层：对这 S_1 个邻居，各自再采样 S_2 个邻居；
- 第 K 层：依此类推。

最终，目标节点 v 的嵌入由约 S_1×S_2×...×S_K 个节点的信息聚合而成（实际因重叠会更少）。典型配置如 K=2, S_1=25, S_2=10，即两层各采样 25 与 10 个邻居，覆盖约 250+ 个节点的信息。

### 2.3 采样的权衡

| 采样规模 | 计算成本 | 信息覆盖 | 嵌入质量 | 适用场景 |
|---------|---------|---------|---------|---------|
| 小（如 [10, 5]） | 低 | 少 | 一般 | 大规模图、快速训练 |
| 中（如 [25, 10]） | 中 | 适中 | 良好 | 通用推荐配置 |
| 大（如 [50, 20]） | 高 | 丰富 | 优秀 | 小图、追求最高质量 |

采样是 GraphSAGE 区别于 GCN 的关键设计。GCN 聚合全部邻居（不做采样），在小图上效果好但难以扩展；GraphSAGE 通过采样实现了可扩展性与归纳能力的统一。

## 三、聚合函数对比

GraphSAGE 的另一核心创新是提出了多种聚合函数（Aggregator），用于把邻居的向量集合聚合为单一向量。不同的聚合函数适合不同的图结构与任务。

| 聚合函数 | 原理 | 优点 | 缺点 | 适用场景 |
|---------|------|------|------|---------|
| Mean（均值） | 对邻居向量取均值 | 简单高效，稳定 | 不区分邻居重要性 | 通用基线，默认选择 |
| GCN（归一化求和） | 按邻居度数归一化的加权求和 | 等价于谱图卷积的一阶近似 | 对度数敏感 | 同质图、与 GCN 对齐 |
| Pool（池化） | 对邻居向量做 MLP 后取 max-pooling | 可学习非线性，表达力强 | 计算成本较高 | 异质图、需区分邻居 |
| Attention（注意力） | 用注意力机制学习邻居权重 | 自适应加权，效果最优 | 计算最复杂 | 重要邻居差异大的场景 |

### 3.1 Mean 聚合

最简单的聚合方式：把采样到的邻居向量取算术平均，再与节点自身向量拼接或相加。它的优点是无参数、计算快、稳定，是 GraphSAGE 的默认聚合器。缺点是所有邻居一视同仁，无法区分"重要邻居"与"噪声邻居"。

### 3.2 GCN 聚合

GCN 聚合器对邻居向量做按度数归一化的加权求和，数学上等价于 GCN（Kipf & Welling）的一阶谱卷积。它考虑了邻居的度数——度数高的邻居权重低（因为它的信息更"分散"）。这种归一化使聚合结果对图的全局结构更敏感。

### 3.3 Pool 聚合

Pool 聚合器先对每个邻居向量施加一个共享的 MLP（多层感知机），再对结果做逐元素 max-pooling。max-pooling 的作用是从所有邻居中选择"最显著"的特征维度，相当于让模型学习"哪些邻居特征最重要"。Pool 聚合的表达能力强于 Mean，但多了一组 MLP 参数需要训练。

### 3.4 Attention 聚合

Attention 聚合器用注意力机制（Attention Mechanism）为每个邻居学习一个权重，再加权求和。这一思路后来被 GAT（Graph Attention Network）独立发展为完整的模型。Attention 聚合能自适应地识别重要邻居，在邻居重要性差异大的场景（如异质图）效果最优，但计算与参数成本最高。

## 四、GraphSAGE 的前向传播算法

GraphSAGE 的每一层执行以下步骤（以第 k 层为例）：

1. **采样**：对目标节点 v，采样其邻居集合 N(v)（规模 S_k）；
2. **邻居聚合**：用聚合函数 AGG_k 把邻居的上一层嵌入 {h_u^{k-1} : u ∈ N(v)} 聚合为单一向量 h_{N(v)}^k；
3. **自身信息融合**：把 v 自身的上一层嵌入 h_v^{k-1} 与聚合向量拼接，再做线性变换与非线性激活：
   $$h_v^k = \sigma\left( W^k \cdot \text{CONCAT}(h_v^{k-1}, h_{N(v)}^k) \right)$$
4. **归一化**：对 h_v^k 做 L2 归一化，防止向量尺度发散。

经过 K 层后，节点 v 的最终嵌入 h_v^K 融合了 K 跳邻域的信息。第 0 层的嵌入 h_v^0 通常是节点的原始特征向量（若无特征则用节点度的 one-hot 编码）。

训练时，GraphSAGE 用监督损失（如交叉熵）或无监督损失（鼓励相邻节点嵌入相近、不相邻节点嵌入远离）端到端地学习所有 W^k 参数。训练完成后，聚合函数的参数固定，可用于任何新节点的嵌入生成。

## 五、归纳学习能力的意义

GraphSAGE 的归纳能力是其相对直推式方法的核心价值，体现在以下几个场景：

### 5.1 动态图与新节点

引文网络新增论文、社交网络新增用户、知识图谱新增实体——这些新节点只需接入图（建立边），即可用已训练的 GraphSAGE 模型生成嵌入，无需重训整图。这在工程上节省了大量计算成本。

### 5.2 跨图泛化

GraphSAGE 学到的是"聚合规则"，可以迁移到完全不同的图上。例如，在一个大学的社交网络上训练的 GraphSAGE 模型，可以直接应用于另一个大学的社交网络生成嵌入（只要节点特征空间一致）。直推式方法做不到这一点——它们只对训练图上的节点有意义。

### 5.3 训练-测试分离

直推式方法在嵌入时会"看到"测试节点（因为整张图都参与计算），存在信息泄漏风险。GraphSAGE 的归纳训练严格区分训练子图与测试子图，更符合机器学习的规范。

## 六、与 Node2Vec 的关键区别

| 对比维度 | Node2Vec | GraphSAGE |
|---------|----------|-----------|
| 学习范式 | 直推式 | 归纳式 |
| 学习对象 | 每个节点的向量 | 聚合函数的参数 |
| 新节点处理 | 无法处理，需重训 | 直接生成嵌入 |
| 节点特征利用 | 不利用 | 必须利用（无特征时用度编码） |
| 聚合方式 | 隐式（游走共现） | 显式（邻域采样 + 聚合函数） |
| 训练成本 | 中（游走 + Word2Vec） | 高（神经网络训练） |
| 跨图能力 | 无 | 有 |
| 参数量 | 少 | 多（每层 W^k） |
| 适用规模 | 中等 | 中等偏大（采样后可扩展） |

一句话总结：Node2Vec 学的是"每个节点长什么样"，GraphSAGE 学的是"如何根据邻居算出节点长什么样"。前者是查表，后者是函数。

## 七、Neo4j GDS 中的 GraphSAGE 调用

GDS 提供了 GraphSAGE 的训练与推理管道。与 FastRP/Node2Vec 直接调用不同，GraphSAGE 需要先训练模型再生成嵌入。

### 7.1 训练 GraphSAGE 模型

```cypher
// 1. 图投影（含节点特征）
CALL gds.graph.project(
  'paperGraph',
  {
    Paper: {
      properties: ['tfidfVector', 'year']
    }
  },
  { CITES: { orientation: 'UNDIRECTED' } }
);

// 2. 训练 GraphSAGE 模型（无监督）
CALL gds.beta.graphSage.train(
  'paperGraph',
  {
    modelName: 'paperSage',
    featureProperties: ['tfidfVector', 'year'],
    embeddingDimension: 128,
    aggregator: 'mean',        // 聚合函数
    sampleSizes: [25, 10],     // 两层采样规模
    epochs: 10,
    learningRate: 0.1,
    batchSize: 512
  }
) YIELD modelName, trainMillis;
```

### 7.2 用训练好的模型生成嵌入

```cypher
// 对图中所有节点生成嵌入（含训练时未见的新节点）
CALL gds.beta.graphSage.write(
  'paperGraph',
  {
    modelName: 'paperSage',
    writeProperty: 'sageEmbedding'
  }
) YIELD nodePropertiesWritten;
```

### 7.3 对新节点做归纳推理

当引文网络新增论文后，只需把新论文接入图（建立引用边、设置特征属性），重新运行 write 过程，模型会自动为新节点生成嵌入，已有节点的嵌入也会更新。无需重新训练模型。

参数说明：

| 参数 | 含义 | 推荐值 |
|------|------|-------|
| embeddingDimension | 嵌入维度 | 128-256 |
| aggregator | 聚合函数 | mean（默认）/ gcn / pool |
| sampleSizes | 采样规模 | [25, 10]（两层） |
| epochs | 训练轮数 | 10-20 |
| learningRate | 学习率 | 0.01-0.1 |
| batchSize | 批大小 | 512 |

## 八、图像领域的应用：新图像归纳分类

GraphSAGE 的归纳能力在图像领域有独特应用，尤其是"新图像分类"场景。

### 8.1 图像相似度图上的归纳嵌入

构建图像相似度图（k 近邻图）后，用部分图像训练 GraphSAGE 模型。当新图像加入时，只需计算它与已有图像的相似度并连边，即可用训练好的模型生成新图像的嵌入，进而分类。这避免了每次新增图像都重训整个嵌入流程。

### 8.2 场景图上的归纳推理

在场景图（Scene Graph）上训练 GraphSAGE 后，可以对新场景（包含新物体）生成场景图嵌入。只要新物体有特征向量（来自 CNN），且与已有物体有关系边，模型就能归纳出合理的场景表示。这是视觉推理走向开放世界（Open-world）的关键能力。

### 8.3 跨数据集泛化

在一个图像数据集上训练的 GraphSAGE 模型，可以迁移到另一个数据集（只要特征空间一致）。例如，在 COCO 数据集的场景图上训练的模型，可应用于 Visual Genome 数据集的场景图。这种跨数据集泛化是直推式方法无法实现的。

## 九、与图书情报领域的关联

GraphSAGE 的归纳能力对图书情报领域意义重大，因为引文网络、本体、用户行为图都是高度动态的。

### 9.1 新论文的即时嵌入

引文网络每天有大量新论文加入。用 GraphSAGE 训练好模型后，新论文只需建立引用关系、提取文本特征，即可即时生成嵌入，用于：

- 新论文的主题分类（预测其所属学科领域）；
- 新论文的相关推荐（找到嵌入相近的已有论文）；
- 新论文的影响力预测（基于结构与特征）。

这一过程无需重训整图，使"实时嵌入"成为可能。相比之下，Node2Vec/FastRP 需要周期性批量重算，无法做到即时。

### 9.2 跨学科引文网络的迁移

在某一学科的引文网络（如计算机科学）上训练的 GraphSAGE 模型，可以迁移到另一学科（如生物学）的引文网络，只要论文特征空间一致（都用相同的文本嵌入方法）。这种迁移学习能力使得"跨学科研究模式分析"成为可能——可以用一个学科学到的模式去理解另一个学科。

### 9.3 动态本体的增量维护

本体与知识图谱持续演进，新概念不断加入。GraphSAGE 可以为新概念即时生成嵌入，支持概念推荐、本体补全等应用，而无需重建整个本体的嵌入。这对大型本体（如 UMLS、MeSH）的维护尤为重要。

### 9.4 用户行为图的实时推荐

图书馆/数据库的用户行为图（用户-资源交互）持续有新用户与新资源加入。GraphSAGE 可以为新用户/新资源即时生成嵌入，驱动实时个性化推荐。新用户只需有几次行为（建立几条边），模型就能生成合理的嵌入，实现"冷启动"友好。

## 十、小结

GraphSAGE 通过"邻域采样 + 可学习聚合函数"的范式，把图嵌入从"查表"升级为"函数推理"，实现了归纳学习能力。它能够处理训练时未见的新节点、迁移到新图、严格分离训练与测试。这些能力使其在动态图、跨图泛化、实时推理等场景中具有直推式方法无法比拟的优势。代价是训练成本更高、参数更多。下一章将深入 GCN 与 GAT，从谱图理论与注意力机制的视角进一步理解图卷积的本质。

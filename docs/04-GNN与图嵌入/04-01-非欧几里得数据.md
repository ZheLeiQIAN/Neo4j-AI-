## 一、引言：为什么数据"形状"很重要

在机器学习（Machine Learning，ML）的发展史上，绝大多数经典模型都隐含着一个基本假设：数据存在于一个规则的、平直的空间中。图像是规则的像素网格，文本是线性的 token 序列，音频是均匀采样的时间序列——这些数据都可以被放置在一个具有平移不变性（Translation Invariance）和明确坐标系的空间里，即欧几里得空间（Euclidean Space）。卷积神经网络（Convolutional Neural Network，CNN）之所以能在图像识别上取得革命性突破，正是因为它巧妙地利用了像素网格的这种规则性。

然而，真实世界中大量关键数据并不具备这种规则结构。社交网络中人与人之间的关注关系、学术文献之间的引用网络、分子的化学键结构、知识图谱中的实体关系——这些数据的"形状"是不规则的、拓扑化的，节点之间没有统一的"邻居数量"，也没有天然的"上下左右"方向。这类数据被称为非欧几里得数据（Non-Euclidean Data）。

本模块要回答的核心问题是：当数据不再是网格、不再是序列，而是一张关系网时，我们如何让机器学习模型"看见"它的结构？这正是图神经网络（Graph Neural Network，GNN）与图嵌入（Graph Embedding）技术兴起的理论动因。

## 二、欧几里得数据 vs 非欧几里得数据

理解非欧几里得数据，最直接的方式是将其与欧几里得数据进行系统对比。

| 对比维度 | 欧几里得数据（Euclidean Data） | 非欧几里得数据（Non-Euclidean Data） |
|---------|------------------------------|--------------------------------------|
| 数据形态 | 规则网格、线性序列 | 不规则图、流形、层级结构 |
| 典型实例 | 图像（像素网格）、文本（token 序列）、音频（时间序列）、表格数据 | 社交网络、知识图谱、引文网络、分子结构、场景图（Scene Graph）、道路网 |
| 空间结构 | 平直空间，具有全局坐标系 | 拓扑空间，无全局坐标系，仅有局部邻接关系 |
| 邻居数量 | 固定且均匀（如图像中每个像素有 8 邻域） | 可变且不均匀（图中节点度数从 0 到数万不等） |
| 平移不变性 | 存在（图像中物体平移后语义不变） | 不存在（图中无统一方向） |
| 卷积定义 | 标准卷积核可直接滑动 | 需重新定义谱图卷积或空域卷积 |
| 典型模型 | CNN、RNN、Transformer | GCN、GAT、GraphSAGE、Node2Vec |
| 数据表示 | 张量（Tensor），形状固定 | 图（Graph），节点数与边数可变 |

需要强调的是，"欧几里得"与"非欧几里得"并非价值判断，而是描述数据所在空间的几何性质。欧几里得数据之所以"好处理"，是因为它的空间结构提供了强先验（Prior）：相邻像素大概率属于同一物体、相邻 token 大概率语义相关。而非欧几里得数据缺少这种先验，模型必须从图结构本身去学习"什么是邻居""邻居有多重要"。

### 2.1 典型非欧几里得数据实例

- **社交网络**：用户为节点，关注/好友关系为边。节点度数服从幂律分布（Power-law Distribution），少数节点（大V）拥有海量邻居。
- **知识图谱**：实体为节点，关系为带类型的边。结构异质（Heterogeneous），不同关系的语义差异巨大。
- **分子结构**：原子为节点，化学键为边。图的结构直接决定分子的化学性质，是药物发现的核心数据。
- **场景图**（Scene Graph）：图像中检测到的物体为节点，物体间的空间/语义关系（"在...上""属于...颜色"）为边。场景图将图像从像素网格转化为关系图，是视觉推理（Visual Reasoning）的桥梁。
- **引文网络**（Citation Network）：论文为节点，引用关系为有向边。是图书情报领域最典型的非欧几里得数据。

## 三、传统 ML/CNN 在非欧几里得数据上的局限

当我们试图把处理欧几里得数据的成熟方法直接搬到非欧几里得数据上时，会遇到一系列根本性困难。这些困难不是工程实现的问题，而是理论层面的不匹配。

### 3.1 无法定义标准卷积核

CNN 的核心机制是卷积核（Convolutional Kernel）在图像上滑动，每个位置都执行相同的局部加权求和。这一操作依赖两个前提：第一，每个位置都有相同数量的邻居（如 3×3 卷积核对应 8 邻域）；第二，邻居可以按固定顺序排列（上、下、左、右）。

在图中，节点的邻居数量是可变的——一个节点可能有 2 个邻居，另一个可能有 5000 个。更重要的是，图的邻居没有自然的排列顺序：节点 A 的邻居 {B, C, D} 并没有"第一个、第二个、第三个"的固有顺序。因此，标准的滑动卷积在图上无法直接定义。

### 3.2 平移不变性缺失

在图像中，"一只猫在左上角"和"一只猫在右下角"应当被识别为同一物体——这就是平移不变性。CNN 通过参数共享（Weight Sharing）天然实现了这一点。

但在图中，"平移"这个概念本身就不成立。图中没有全局坐标系，节点的"位置"只由其与他人的关系定义。你不能把一个子图"向右移动三个节点"，因为图中没有"右"。这意味着 CNN 那种基于空间位置共享参数的机制，在图上失去了直接对应物。

### 3.3 结构上下文的丢失

传统 ML 的另一局限在于：它倾向于把每个样本视为独立同分布（i.i.d.）的数据点，而忽略了样本之间的关系结构。如果把社交网络中的每个用户作为一行表格数据，用户之间的关系就被丢弃了；如果把引文网络中的每篇论文作为独立样本，论文间的引用影响就消失了。

然而，正是这种关系结构承载着非欧几里得数据的核心语义。一篇论文的重要性不仅取决于其自身内容，更取决于谁引用了它、它引用了谁、它在引用网络中处于什么位置。丢弃结构等于丢弃信息。

## 四、核心论点深度解读

本系列讲座的核心论点可以凝练为一句话：

> **传统机器学习模型难以捕捉非欧几里得空间中的关键结构上下文（Structural Context）。**

这句话包含三个需要深度解读的关键概念。

**"传统机器学习模型"** 指的是以 i.i.d. 假设为基础、以表格/张量为输入的模型族，包括逻辑回归、随机森林、SVM，以及最初为欧几里得数据设计的 CNN/RNN。这些模型在设计时并未考虑数据点之间的关系结构。

**"非欧几里得空间"** 指的是数据点之间存在不规则、可变、无方向性的连接关系。这类空间中，两点之间的"距离"不再是直线距离，而是可能涉及多条路径的图距离（Graph Distance）；"邻域"不再是固定大小的窗口，而是由图拓扑决定的可变集合。

**"关键结构上下文"** 是这句话的重心。结构上下文指的是：一个节点的语义不仅由其自身属性决定，还由它在图中的位置、它的邻居是谁、邻居们如何相互连接共同决定。例如：

- 在引文网络中，一篇论文的"学术影响力"由引用它的论文数量与质量（PageRank 值）共同决定；
- 在社交网络中，一个用户的"兴趣画像"不仅看其发布内容，还看其好友关注什么（同质性 Homophily）；
- 在分子图中，一个原子的"化学反应性"取决于它与哪些原子成键、成几条键、键角如何（结构等价 Structural Equivalence）。

传统模型之所以"难以捕捉"这些上下文，根本原因在于它们的输入接口（定长特征向量）无法容纳可变大小的邻域信息。即使手工构造一些图统计特征（如度数、聚类系数），也只能捕捉非常局部的结构，无法表达多跳邻域的递归影响。图神经网络正是为突破这一瓶颈而生：它通过消息传递（Message Passing）机制，让每个节点递归地聚合其邻域信息，从而把结构上下文"编织"进节点的向量表示中。

## 五、图作为非欧几里得数据的统一表示

非欧几里得数据形态各异——社交网络、分子、知识图谱、场景图看似毫无共性。但它们都可以被统一抽象为**图**（Graph）。图由节点集合 V 与边集合 E 组成，记作 G = (V, E)。这一抽象的强大之处在于：

1. **统一的存储与查询**：无论数据来源如何，都可以用属性图模型存储于 Neo4j，用 Cypher 查询；
2. **统一的算法库**：GDS 提供的算法（中心性、社区检测、嵌入）对任何图都适用；
3. **统一的机器学习接口**：GNN 框架把所有非欧几里得数据都视为图，用同一套消息传递机制处理。

这种统一性使得图数据库成为非欧几里得数据的"通用容器"，而图神经网络成为处理非欧几里得数据的"通用模型"。

## 六、图数据的两种特征

要理解 GNN 如何学习图，必须区分图数据中的两类特征。

### 6.1 节点特征（Node Features）

节点特征是附着在节点上的属性向量，描述节点"自身是什么"。例如：

- 社交网络中，用户的年龄、性别、兴趣标签向量；
- 引文网络中，论文的 TF-IDF 向量或BERT 嵌入；
- 分子图中，原子的元素类型、电荷、杂化方式（one-hot 编码）；
- 图像场景图中，物体的类别嵌入、颜色、位置坐标。

节点特征是"内容侧"的信息，它回答"这个节点本身具有什么属性"。

### 6.2 结构特征（Structural Features）

结构特征是隐含在图拓扑中的信息，描述节点"在图中处于什么位置、与谁连接"。结构特征并不直接存储为属性，而是需要通过图算法提取。例如：

- 度（Degree）、入度、出度；
- 聚类系数（Clustering Coefficient）；
- 介数中心性（Betweenness Centrality）、PageRank；
- 所在社区的标签（社区检测结果）；
- 邻域的构成（邻居的类型分布、邻居的属性聚合）。

结构特征是"关系侧"的信息，它回答"这个节点在网络中扮演什么角色"。

### 6.3 两类特征的融合

GNN 的核心创新在于：它不是分别处理这两类特征然后拼接，而是在每一层中**让节点特征与结构特征交互**。具体而言，每一层 GNN 都做两件事：（1）聚合邻居的节点特征（利用结构信息）；（2）将聚合结果与节点自身特征结合（保留内容信息）。经过多层堆叠，节点的表示既包含自身的属性，也融合了 k 跳邻域的结构上下文。

这种"内容+结构"的融合正是 GNN 区别于传统 ML 的本质。传统 ML 只能用节点特征，GNN 则把结构特征以可学习的方式注入表示中。

## 七、与图书情报领域的关联

非欧几里得数据的概念对图书情报（Library and Information Science，LIS）领域具有特殊的意义，因为该领域的核心数据几乎都是非欧几里得的。

### 7.1 引文网络是非欧几里得数据的典型

引文网络（Citation Network）是图书情报学研究的基础数据结构。论文之间的引用关系构成一张有向无环图（DAG），其中：

- 节点 = 论文（携带标题、摘要、作者、年份等节点特征）；
- 有向边 = 引用关系（A 引用 B）；
- 结构特征 = 被引次数、h 指数、PageRank、引用路径。

经典的文献计量学指标（影响因子、被引频次）本质上是对引文网络结构特征的粗粒度提取。而 GNN 可以在这张图上学习更精细的节点嵌入，用于论文分类、学者聚类、研究前沿识别、论文影响力预测等任务。

### 7.2 知识组织体系的图本质

叙词表（Thesaurus）、本体（Ontology）、分类法（Classification Scheme）在数据结构上都是图：

- 叙词表中的"用、代、属、分、参"关系构成一张语义图；
- 本体中的类层次（subclass-of）与属性关系是典型的有向图；
- 《中图法》《杜威十进分类法》的分类树是树状图的特例。

把这些知识组织工具建模为非欧几里得数据并用 GNN 处理，可以实现概念的自动对齐（Alignment）、相似概念发现、分类号推荐等智能化服务。

### 7.3 用户与资源的交互图

图书馆的借阅记录、数据库的检索日志天然构成二部图（Bipartite Graph）：用户节点与资源节点通过"借阅/检索"边连接。这张图是推荐系统（Recommendation System）的核心数据，而基于图的协同过滤（Graph-based Collaborative Filtering）正是 GNN 在该场景的应用。

理解非欧几里得数据，是图书情报领域从"文献计量"走向"图智能"的认知前提。后续章节将系统介绍如何在 Neo4j 中用图嵌入与 GNN 算法处理这类数据。

## 八、本模块学习路径

本模块将按照以下路径展开：

1. **图嵌入技术总览**：建立对各类嵌入方法的整体认知；
2. **Node2Vec 与 DeepWalk**：从随机游走入门的经典方法；
3. **FastRP**：工业级高效嵌入算法；
4. **GraphSAGE**：归纳式图神经网络的代表；
5. **GCN 与 GAT**：谱图与注意力视角下的图卷积；
6. **Neo4j 中的 GNN 实践**：在 GDS 中的工程落地；
7. **GDS 机器学习管道**：从嵌入到下游任务的完整流程。

每一节都会回到本章的核心论点：如何让模型"看见"非欧几里得数据的结构上下文。

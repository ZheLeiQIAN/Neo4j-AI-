## 一、FastRP 的设计动机

Node2Vec 与 DeepWalk 虽然理论优雅、效果良好，但在工业级规模图上存在一个显著瓶颈：随机游走采样 + Word2Vec 训练的计算成本随图规模线性甚至超线性增长。一张千万节点的图，Node2Vec 可能需要数十分钟甚至数小时才能完成嵌入。

研究者进一步发现一个理论事实：DeepWalk/Node2Vec 的随机游走 + Skip-gram 训练，在数学上等价于对节点共现矩阵（近似 PMI 矩阵）做隐式矩阵分解（Matrix Factorization）。既然如此，为什么不直接显式地分解这个矩阵，跳过耗时的游走与训练？

FastRP（Fast Random Projection，快速随机投影）正是沿着这一思路发展出的算法（Chen 等，2019）。它用稀疏随机投影（Sparse Random Projection）近似矩阵分解，在不牺牲嵌入质量的前提下，把速度提升了**一到两个数量级**。在 Neo4j GDS 中，FastRP 是生产环境中最受推荐的节点嵌入算法之一。

## 二、FastRP 的数学原理

理解 FastRP 需要把握三个关键步骤：邻接矩阵的幂级数、随机投影近似、特征组合。

### 2.1 邻接矩阵的幂级数与多跳邻域

设 A 为图的（归一化）邻接矩阵。A 的第 k 次幂 A^k 的元素 (A^k)_{ij} 近似反映了节点 i 与 j 之间长度为 k 的游走路径数量。因此，A^k 编码了 k 跳邻域的结构信息：

- A^1：1 跳邻域（直接邻居）；
- A^2：2 跳邻域（朋友的朋友）；
- A^3：3 跳邻域；
- ……

把不同跳数的邻域信息组合起来，可以得到对节点结构上下文的全面刻画。FastRP 的目标矩阵是这些幂的加权组合：

$$M = \sum_{k=1}^{L} w_k \cdot A^k$$

其中 L 是最大跳数（对应 iterationWeights 的长度），w_k 是第 k 跳的权重。

### 2.2 随机投影近似

直接计算 M 并分解其维度为 |V|×|V|，在千万节点规模下不可行（需要存储 10^14 个元素的矩阵）。FastRP 的关键创新是用随机投影（Random Projection）降维。

Johnson-Lindenstrauss 引理保证：一个高维空间中的点集，可以通过随机投影到低维空间，而近似保持点对之间的距离。FastRP 生成一个稀疏随机矩阵 R（维度 |V|×d，d 为嵌入维度），其每个元素以一定概率取 {−1, 0, +1} 之一（稀疏化以加速计算）。于是嵌入矩阵为：

$$E = M \cdot R = \sum_{k=1}^{L} w_k \cdot A^k \cdot R$$

关键在于，A^k·R 可以递归地计算：先算 A·R（1 跳），再用结果乘 A 得到 A^2·R（2 跳），依此类推。每次只需要做一个稀疏矩阵乘法，复杂度远低于显式构造 M。

### 2.3 多跳特征组合与归一化

不同跳数的特征被加权组合后，再做 L2 归一化，得到最终的节点嵌入。权重的选择决定了不同跳数邻域的相对重要性。GDS 默认的权重配置通常对前几跳赋予较高权重，反映了"近邻比远邻更重要"的直觉。

## 三、与 Node2Vec 的对比优势

FastRP 与 Node2Vec 在嵌入质量上可比，但在工程性能上有显著差异。

| 对比维度 | FastRP | Node2Vec |
|---------|--------|----------|
| 计算速度 | 极快（矩阵乘法，并行性好） | 较慢（游走采样 + Word2Vec 训练） |
| 可扩展性 | 优秀，支持千万级节点 | 受限于游走数量与训练时长 |
| 确定性 | 确定性（给定随机种子，结果可复现） | 有随机性（游走采样随机） |
| 参数复杂度 | 少（维度、权重、迭代次数） | 多（p、q、游走长度、窗口等） |
| 结构平衡控制 | 通过权重控制各跳贡献 | 通过 p、q 控制同质/结构等价 |
| 利用节点特征 | 可融合（featureProperties） | 不支持 |
| 归纳能力 | 通过投影矩阵可部分支持 | 不支持 |
| 理论基础 | 显式矩阵分解 + 随机投影 | 隐式矩阵分解（游走） |

FastRP 的核心优势是**速度与确定性**。在相同硬件上，FastRP 通常比 Node2Vec 快 10 到 100 倍。对于需要频繁重算嵌入的动态图场景（如每日更新的引文网络），这种速度差异是决定性的。

FastRP 的另一优势是**可融合节点特征**。通过 featureProperties 参数，FastRP 可以把节点的属性向量纳入嵌入计算，使嵌入同时反映结构信息与内容信息——这是 Node2Vec 做不到的。

## 四、参数配置与调优

FastRP 在 GDS 中的参数较少但语义重要，调优空间集中在嵌入维度与迭代权重上。

| 参数 | 含义 | 推荐值 | 调优建议 |
|------|------|-------|---------|
| embeddingDimension | 嵌入向量维度 | 256（默认） | 大图用 256-512，小图 128 即可 |
| iterationWeights | 各跳权重列表，长度决定最大跳数 L | [0.0, 1.0, 1.0]（默认，2-3 跳） | 增大权重数覆盖更远邻域；权重递减模拟衰减 |
| featureProperties | 融合的节点特征属性 | 无（默认） | 有内容特征时填入属性名列表 |
| featureDimension | 特征属性的维度 | 无 | 与 featureProperties 配合 |
| propertyRatio | 结构与特征的混合比例 | 0.5 | 0=纯结构，1=纯特征，0.5=均衡 |
| normalizationStrength | L2 归一化强度 | 1.0 | 建议保持默认 |
| randomSeed | 随机种子 | 无 | 设定以保证可复现 |

### 4.1 iterationWeights 的调优要点

iterationWeights 是 FastRP 最关键的可调参数。它是一个浮点数列表，长度 L 决定考虑的最大跳数，第 i 个值是第 i 跳的权重。常见配置：

- `[0.0, 1.0, 1.0]`：忽略 0 跳（自身），等权考虑 1-2 跳（默认）；
- `[0.0, 1.0, 1.0, 1.0]`：扩展到 3 跳，覆盖更远邻域；
- `[0.0, 1.0, 0.5, 0.25]`：权重随跳数衰减，模拟"远邻影响递减"；
- `[1.0, 1.0, 1.0]`：包含 0 跳（自身特征），适合需要保留节点自身信息的场景。

跳数越多，计算越慢但覆盖越广。实践中 2-4 跳通常足够，更多跳数的边际收益递减。

### 4.2 特征融合的调优

当使用 featureProperties 时，propertyRatio 控制结构与特征的混合比例：

- propertyRatio = 0：纯结构嵌入，等价于不用特征；
- propertyRatio = 1：纯特征嵌入，忽略结构；
- propertyRatio = 0.5：结构与特征各占一半（推荐起点）。

若节点特征质量高（如经过预训练的 BERT 嵌入），可适当提高 propertyRatio；若特征稀疏或噪声大，则降低比例以依赖结构。

## 五、Neo4j GDS 中的 FastRP 调用

以下是一个生产级的 FastRP 调用示例，展示纯结构嵌入与特征融合两种用法。

### 5.1 纯结构嵌入

```cypher
// 1. 图投影
CALL gds.graph.project(
  'citationGraph',
  'Paper',
  { CITES: { orientation: 'UNDIRECTED' } }
);

// 2. 运行 FastRP，纯结构
CALL gds.fastRP.write(
  'citationGraph',
  {
    writeProperty: 'fastrpStruct',
    embeddingDimension: 256,
    iterationWeights: [0.0, 1.0, 1.0, 0.5],
    randomSeed: 42,
    concurrency: 8
  }
) YIELD nodePropertiesWritten, computeMillis;

// 3. 验证
MATCH (p:Paper {id: 'paper-001'})
RETURN p.fastrpStruct AS embedding;
```

### 5.2 融合节点特征

```cypher
// 1. 图投影（假设论文节点有 bertEmbedding 属性，维度 768）
CALL gds.graph.project(
  'citationGraphFeat',
  'Paper',
  { CITES: { orientation: 'UNDIRECTED' } },
  {
    nodeProperties: ['bertEmbedding']
  }
);

// 2. 运行 FastRP，融合 BERT 特征
CALL gds.fastRP.write(
  'citationGraphFeat',
  {
    writeProperty: 'fastrpHybrid',
    embeddingDimension: 256,
    iterationWeights: [0.0, 1.0, 1.0],
    featureProperties: ['bertEmbedding'],
    featureDimension: 768,
    propertyRatio: 0.5,        // 结构与特征各半
    randomSeed: 42
  }
) YIELD nodePropertiesWritten, computeMillis;
```

融合后的嵌入既反映引用结构相似性，又反映内容语义相似性，通常在下游任务（论文分类、推荐）上表现优于纯结构或纯特征嵌入。

## 六、直推式与归纳式配置

FastRP 本质上是直推式方法——它需要看到完整图才能计算嵌入。但 GDS 提供了一种近似归纳能力：通过保留训练阶段生成的随机投影矩阵 R，可以对训练后加入的新节点生成嵌入。

具体而言，新节点 u 的嵌入可计算为：

$$e_u = \sum_{k=1}^{L} w_k \cdot (A^k \cdot R)_u$$

即把新节点接入图后，重新计算它到各跳邻居的连接，并与已有的 R 相乘。这种方式不需要重新训练整个图，只需对新节点的局部邻域做计算，因此可以近似地处理新节点。但需注意，这种"归纳"能力是有条件的——新节点必须能接入已有图，且投影矩阵 R 不变。

GDS 中通过 `mutateProperty` 与 `writeProperty` 的配合，可以实现增量更新：对已有节点保留嵌入，仅对新增节点计算。

## 七、性能对比表

以下是基于公开基准与工程经验的典型性能对比（图规模约 100 万节点）。

| 算法 | 典型耗时 | 嵌入质量（节点分类 F1） | 可扩展性 | 确定性 | 特征融合 |
|------|---------|----------------------|---------|--------|---------|
| FastRP | 10-30 秒 | 0.78-0.82 | 极强 | 是 | 支持 |
| Node2Vec | 10-30 分钟 | 0.79-0.83 | 中 | 否 | 不支持 |
| GraphSAGE | 30-60 分钟 | 0.82-0.88 | 中 | 否 | 支持 |
| HashGNN | 5-15 秒 | 0.75-0.80 | 强 | 是 | 支持 |

可以看到，FastRP 在速度上有压倒性优势，质量与 Node2Vec 相当。GraphSAGE 质量最高但耗时最长。在"速度/质量"比上，FastRP 是性价比最高的选择，因此常被用作生产环境的默认嵌入算法。

## 八、与图书情报领域的关联

FastRP 在图书情报场景的优势尤为突出，因为引文网络、本体、用户行为日志通常规模巨大且持续更新。

### 8.1 大规模引文网络的快速嵌入

Web of Science、Scopus、CNKI 等数据库的引文网络包含数千万甚至上亿篇论文。在这样的规模上，Node2Vec 几乎不可行，而 FastRP 可以在合理时间内完成嵌入。这使得"对全量引文网络做结构嵌入"从理论可能变为工程可行，为大规模文献计量分析提供了新工具。

### 8.2 融合文本与结构的混合嵌入

FastRP 的特征融合能力对图书情报特别有价值。论文节点通常有丰富的文本特征（标题、摘要的 BERT 嵌入），FastRP 可以把文本语义特征与引用结构特征融合到同一向量中。这种混合嵌入比纯文本或纯结构嵌入更能准确刻画论文的"学术身份"：

- 纯文本嵌入：两篇方法相同但表述不同的论文可能距离远；
- 纯结构嵌入：两篇引用结构相似但内容无关的论文可能距离近；
- 混合嵌入：兼顾内容与方法相似性，更符合学术直觉。

### 8.3 本体与叙词表的快速向量化

大型本体（如 UMLS、Agrovoc）包含数十万概念，FastRP 可以快速为每个概念生成结构嵌入，支持概念相似度计算、本体对齐、概念推荐等应用。对于需要频繁重建嵌入的本体维护工作流，FastRP 的速度优势意味着可以在分钟级完成重算，而非小时级。

### 8.4 确定性带来的可复现性

学术研究对结果可复现性要求高。FastRP 的确定性（设定 randomSeed 后结果可复现）使其更适合作为研究中的标准嵌入方法——同一张图、同一组参数，永远得到相同的嵌入，便于对比实验与论文发表。Node2Vec 的随机性则需要多次运行取平均才能稳定结果。

---

## 九、小结

FastRP 通过"矩阵分解 + 随机投影"的思路，把图嵌入的计算成本降低了两个数量级，同时保持了与 Node2Vec 相当的嵌入质量。它的确定性、可扩展性与特征融合能力，使其成为 Neo4j GDS 中生产环境的首选嵌入算法。对于图书情报领域的大规模图数据，FastRP 是让"图嵌入"从理论走向日常工程实践的关键工具。下一章将介绍 GraphSAGE，它是另一种范式——归纳式图神经网络的代表。
